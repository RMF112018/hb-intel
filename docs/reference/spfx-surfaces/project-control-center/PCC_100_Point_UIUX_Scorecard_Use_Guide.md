## 1. Purpose

This guide explains how to apply the PCC 100-point UI/UX Mold Breaker scorecard and how to interpret evidence generated by the live PCC Playwright suite.

The scorecard defines the standard. This guide defines how to use the standard.

## 2. Key Rule

The final 100-point score is not automated.

Playwright-generated evidence supports expert scoring. It does not calculate the final score, does not mark EVs as finally captured, and does not mark hard stops as passed or failed.

## 3. EV Taxonomy Interpretation

The canonical PCC evidence taxonomy includes `EV-001` through `EV-134`.

The current Playwright evidence suite supports this subset:

```text
EV-037..EV-106
EV-125..EV-134
```

This gives the Playwright suite **80 supported EV IDs**.

The remaining EV IDs remain part of the scorecard evidence model, but they are manual, documentary, operator-supplied, hybrid, or expert-review evidence unless later added to the Playwright registry.

## 5. Manual / Documentary EV Treatment

Capture the following groups through audit reports, planning docs, source-review records, expert notes, or operator-supplied closeout files:

```text
EV-001..EV-036
EV-107..EV-124
```

These items may include:

- Governance review notes.
- Repo-truth source review.
- Architecture/documentation review.
- Manual score rationale.
- Expert findings.
- Leadership acceptance.
- Exception logs.
- Phase 4 readiness statement.

## 6. Playwright Output Handling

Playwright outputs should be treated as:

- Evidence-support artifacts.
- Traceability artifacts.
- Expert-review inputs.
- Operator-reviewed evidence packages.

They should not be treated as:

- Final score.
- Final Phase 4 readiness approval.
- Final hard-stop disposition.
- Automatic evidence capture completion.
- Production deployment approval.

## 7. Required Live Evidence Closeout

Every full evidence run should be closed using:

```text
docs/architecture/evidence/pcc-live/README.md
docs/architecture/evidence/pcc-live/PCC_Live_Evidence_Closeout_Template.md
```

The closeout must identify:

- Run ID.
- Package version.
- Commit SHA.
- Tenant page.
- Environment variables used, excluding secrets.
- Commands executed.
- Artifact directories.
- Completed lanes.
- Self-skipped lanes.
- Operator-pending lanes.
- Evidence scrub/review status.
- Known failures.
- Scorecard/hard-stop impact.

## 8. Recommended Scoring Workflow

1. Confirm canonical scorecard filename references.
2. Review governing docs and Mold Breaker studies.
3. Run or review Playwright evidence.
4. Complete manual/documentary EV records.
5. Complete evidence index.
6. Review hard-stop worksheet.
7. Assign expert pillar scores.
8. Record rationale.
9. Record residual risks.
10. Issue Phase 4 readiness statement.

## 9. Evidence Sufficiency Rules

| Condition | Scoring Treatment |
|---|---|
| Playwright output exists but is not operator-reviewed | Partial support only |
| Screenshot exists but source/runtime is missing | Partial support only |
| Source exists but hosted evidence is missing | Partial support only |
| Accessibility artifacts exist but manual review is incomplete | Partial support only |
| EV is manual/documentary and not completed | Evidence gap |
| Hard-stop evidence is incomplete | Blocks Phase 4 readiness |
| Final score generated without expert review | Invalid score |
