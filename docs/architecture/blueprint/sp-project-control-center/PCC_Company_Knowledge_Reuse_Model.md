# PCC Company Knowledge Reuse Model

## 1. Purpose

Define governed reuse of active and closed project knowledge for future pursuits, estimating, operations, warranty prevention, and executive learning.

## 2. Knowledge Reuse Use Cases

- Comparable project selection during pursuit.
- Estimating reference for assumptions and variances.
- Operations planning based on prior blockers/outcomes.
- Warranty prevention based on historical claims.
- Executive pattern review across authorized projects.

## 3. Comparable Project References

Comparable references should include scope profile, delivery context, outcome summaries, and confidence/evidence posture.

## 4. Estimating Historical References

Authorized users should review historical assumptions, exclusions, alternates, and realized outcomes.

## 5. Lessons Learned Feedback Loop

Closed-project lessons should be linked to future pursuit and estimating contexts through governed references.

## 6. Subcontractor/Vendor Performance Feedback

Vendor/subcontractor outcome context should be permission-filtered and evidence-backed before reuse in future decisions.

## 7. Warranty Claim Pattern Feedback

Warranty claim patterns should inform future scope review, product decisions, and risk controls where evidence supports reuse.

## 8. Cross-Project Security Filtering

Knowledge reuse must enforce role, project, confidentiality, and legal boundaries.

## 9. Closed-Project Reference Mode

Closed projects should support a read-focused reference mode with strict access controls and provenance visibility.

## 10. MVP vs Later-Phase Timing

- MVP: doctrine and documentation alignment.
- Later phase: index/query patterns, reuse scoring, and governed cross-project retrieval flows.

## 11. Guardrails

- No unrestricted cross-project browsing.
- No source ownership reassignment.
- No use of unverified claims as authoritative fact.
- No exposure of restricted executive/financial/HR or privileged legal content.

## Conceptual Record Definitions

### `ProjectKnowledgeReference`

A governed reference record linking lifecycle-relevant project knowledge to authorized future use.

### `CrossProjectReference`

A permission-filtered reference from one project context to another with traceable rationale.

### `LessonLearnedReference`

A structured lesson entry linked to source evidence and downstream applicability context.

### `ComparableProjectReference`

A comparable-project mapping record with declared basis, scope similarity factors, and confidence posture.
