## Apply-Gate Summary

The apply gate is a single pure entry point — `evaluateApplyGate(request, options?)` — that takes a frozen `ApplyGateRequest` and returns a frozen `ApplyGateDecision` with one of three statuses:

- `blocked` — at least one rule failed; `errors` enumerates the violations.
- `not-executable-in-this-step` — request is well-formed and clean, but operator approval has not yet transitioned to `approved-for-non-production`.
- `ready-for-non-production-apply` — request is well-formed, the manifest mutation gate is locked, the proof artifact validates, all manifest scans pass, all 14 object family slots are populated, the target tenant is non-production, the rollback posture is declared, and the operator has approved with `non-production-only` scope.

Every decision — blocked or ready — emits sha256 hashes (`manifestHash`, `proofArtifactHash`, `proofMarkdownHash`) and the literal flags `nonProductionOnly: true` and `tenantMutationPerformed: false`, plus a `notExecutableReason` recording that no executor exists in this step.

The validator never weakens its rules; the only test fix during execution corrected a public-export scan that false-positived on the path-segment `apply-gate`, by narrowing the scan to identifier portions only.

## Executor Adapter Boundary Summary

The future executor adapter will:

- live under `backend/functions/` (no code created in this step),
- consume an `ApplyGateRequest` whose decision evaluates to `ready-for-non-production-apply`,
- additionally verify manifest immutability, regenerated proof byte-match, locked mutation gate, non-production scope, rollback posture, and operator approval scope inside its own process,
- be the only future surface that may call Graph or PnP,
- never re-derive plans inside the executor (read frozen manifest only).

`@hbc/project-site-provisioning` itself remains no-mutation. SPFx must not call the executor directly; any future SPFx-originated apply request must be routed through an approved admin/control-plane flow. No Procore path is introduced.

## Mutation / No-Execution Confirmation

- `evaluateApplyGate` is pure: no I/O, no filesystem write, no network, no tenant-aware operation.
- Manifest mutation gate remains compile-time and runtime locked (`mutationLocked: true`, `liveMutationAllowed: false`, `requiresHumanApproval: true`).
- Proof artifact's literal flags (`dryRunOnly: true`, `tenantMutationAllowed: false`) are preserved.
- Apply-gate decision's literal flags (`nonProductionOnly: true`, `tenantMutationPerformed: false`) are present on every outcome.
- `src/apply-gate/**` contains no `@pnp/`, `@azure/`, `@microsoft/sp-`, `spHttpClient`, or `GraphClient` reference. The existing source-import discipline scan continues to pass without exclusion expansion.
- Public-export discipline scan continues to pass; new exports use validation/gating verbs (`evaluateApplyGate`, `validateApplyGateRequest`, `hashJsonProofArtifact`, `hashMarkdownProofArtifact`); no live-operation verb names are exposed.

## Non-Production-Only Confirmation

- `request.environment === 'production'` → blocked.
- `request.targetTenant.targetEnvironment !== 'non-production'` → blocked.
- `operatorApproval.approvalStatus === 'approved-for-non-production'` requires `approvalScope === 'non-production-only'`; otherwise blocked.
- Approval-state shape rejects status values outside the four-enum allow-list.
- The contract has no path to a "production-ready" decision; the only ready status is `ready-for-non-production-apply`.
