# Phase 0 Step 1 — Consistency Check Register

This register records the formal consistency assessment performed during the Phase 0 Step 1 audit. It evaluates the Standard Project Site Template Contract and the HB Project Control Center Target Architecture Blueprint against the named consistency rules. Each row uses one of the four allowed results: `Pass`, `Pass with Note`, `Fail`, or `Not Verifiable`. A `Pass with Note` row indicates the rule passes against the governing architecture but carries a schema-extraction representation gap that must be closed in Phase 1; it is not a defect in the architecture and does not block Phase 1 entry. A `Fail` row would block Phase 1 entry. The register, as authored, contains **zero `Fail` rows**.

Anti-regression tokens (`preconstruction_only`, `warranty_closeout`, `Archived`-as-stage, `active_construction`-as-ProjectType) appear in this register **only inside anti-regression rule statements**, never as valid enum members.
