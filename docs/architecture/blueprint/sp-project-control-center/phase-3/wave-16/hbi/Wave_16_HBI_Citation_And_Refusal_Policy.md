# Wave 16 HBI Citation And Refusal Policy

## Purpose

Define Wave 16 HBI operating boundaries for settings surfaces, including allowed actions, refusal conditions, and citation requirements.

## Allowed HBI Behaviors

- Explain settings posture and ownership boundaries.
- Summarize validation/audit outcomes with redaction preserved.
- Cite authoritative sources for settings context.
- Explain disabled actions and blocked paths.
- Draft change-request justification text.

## Refused HBI Behaviors

- Change a setting directly.
- Approve or reject workflow items.
- Bypass approval or admin verification.
- Expose or infer secret values.
- Mutate permissions or security ownership.
- Override source-of-record ownership constraints.
- Fabricate validation success or audit outcomes.

## Citation and Lineage Policy

- Responses must anchor to canonical Wave 16 docs and schema references where applicable.
- HBI summaries must preserve source lineage and confidence boundaries.
- Where data is unavailable or restricted, refuse safely with reasoned guidance.

## No-Authority Rule

HBI is explain/summarize/cite only; it has no command authority over settings, approvals, permissions, or tenant/platform state.

## Implementation Boundary

This document defines documentation governance only and does not authorize runtime execution changes.
