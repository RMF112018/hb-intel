# Wave 14 HBI Guardrails and Audit Model

## HBI Guardrail

HBI may summarize and cite evidence only. HBI has no decision authority, no checkpoint approval authority, no waiver authority, and no override authority.

## Reviewer/Approver Distinction

HBI is never a reviewer, approver, admin verifier, or command actor. Any attempted HBI decision action must be refused and audited.

## Audit Model Anchor

Wave 14 governs audit semantics for checkpoint queue actions, route-step transitions, terminal decisions, stale-source holds, supersession events, and decision history.

## Non-Authorization

No runtime AI-action execution, source-system writeback, tenant mutation, or backend command execution is authorized.
