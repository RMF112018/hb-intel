# 02 — Homepage Integration Assessment

## Headline conclusion
The homepage shell integration is technically governed, but strategically weak for the requested target state.

The shell is not breaking SafetyFieldExcellence by accident.
It is governing it in a way that treats it as a **secondary supporting strip** rather than a flagship safety / field-intelligence module.

## 1. What the shell gets right
- Placement is handled through explicit shell contracts rather than ad hoc JSX.
- Safety has a named occupant id and zone wrapper.
- There is a declared shell-fit model.
- The shell can stack when unsafe.
- Presets make homepage composition inspectable and governable.

This is a good foundation.

## 2. What the shell gets wrong for this surface

### A. Wrong semantic neighborhood
Allowing `safety-field-excellence` inside `communications-newsroom` blurs the module’s identity.
It visually and semantically encourages Safety to behave like supporting editorial content.

### B. Wrong prominence ceiling
`prominenceCeiling: 'supporting'` hardcodes the wrong ambition for a surface that is supposed to communicate urgency, field relevance, and “what matters now.”

### C. Wrong flagship default
The default preset places Safety beside Company Pulse in the minor slot.
That makes Safety feel subordinate to newsroom content.

### D. Optimistic compact contract
The shell claims Safety can pair down to ~320px and supports compact mode.
The application does not expose a correspondingly sophisticated compact/minimal model.
The result is shell-legality without true product credibility.

## 3. Effect on first-screen homepage experience
The homepage overlay doctrine requires the first shell lane to deliver meaningful value and insists that hosted surfaces only pair when actual slot width supports a premium nested experience.

The current default composition gives:
- Row 1 operational spotlight + recognition
- Row 2 safety + newsroom

That means Safety is visually delayed and semantically softened.
It is not helping lead the homepage’s operational awareness story.

## 4. Strategic conflict inside the repo
The repo shows competing ideas about Safety’s role:
- the reference composition groups it with other operational awareness modules
- the operations/safety preset brings it forward
- the default flagship preset still demotes it into a newsroom pair

This strategic drift needs to be closed.
A world-class outcome cannot be built on an unresolved identity question.

## 5. Homepage integration direction
The shell should treat SafetyFieldExcellence as one of the following, explicitly:

### Option A — Operational co-anchor
Pair it with Project Portfolio Spotlight in a true first-lane operations row, with a shell contract that recognizes both as operational anchors.

### Option B — Full-width operational strip
Give it its own full-width operational band directly beneath the first lane so it can summarize field-critical signals without minor-slot compression.

### Option C — Hybrid anchor
Keep the project spotlight as first-lane primary anchor and make Safety the first full-width band immediately beneath it, before newsroom content.

For the current target, Option C is the safest high-quality move unless the safety surface is rebuilt into a much stronger co-anchor product.

## 6. Recommendation
Do not leave SafetyFieldExcellence as a newsroom-adjacent minor passenger.
Reclassify it in the shell and update the default preset accordingly.
