# 06 — Recommended Implementation Waves

## Wave 01 — Structural UX rebuild (must happen first)

### Objective
Move the Safety app from “functional baseline” to “credible premium operational workspace.”

### Scope
1. Workspace masthead + navigation rebuild
2. Correct `WorkspacePageShell` state-model adoption
3. Safety-specific primitive family
4. Upload / Periods / Inspections / Review / Detail surface recomposition
5. Responsive contract and compact-state behavior
6. Incidents information-architecture correction

### Why first
These are the blockers that keep the app far below flagship quality even if styling is improved later.

### Expected result
- serious first-screen impression
- meaningful layout hierarchy
- no ambiguous loading/empty states
- better route consistency
- credible tablet/phone behavior

## Wave 02 — Flagship hardening and closure proof

### Objective
Convert the rebuilt surface into an evidence-backed, doctrine-compliant closure candidate.

### Scope
1. Accessibility hardening
2. Focus/keyboard/touch refinement
3. Premium motion and affordance polish
4. Expanded route-state and breakpoint tests
5. Hosted screenshot matrix
6. Package verification / packaged-truth evidence

### Why second
The app should not lock proof artifacts around a weak structural baseline.

### Expected result
- higher confidence acceptance package
- resilient hosted behavior
- more credible flagship score
