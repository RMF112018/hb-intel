# 06 — Recommended Implementation Waves

## Wave 01 — Shell, theme, navigation, and page-contract correction
### Objective
Stop shipping the current wrong posture and bring the Safety app onto a credible product foundation.

### Included work
1. Theme/mode decision and enforcement
2. Header/nav rebuild
3. `WorkspacePageShell` contract compliance across all pages
4. Removal of content-zone direct action anti-patterns
5. Basic host/runtime proof capture

### Exit criteria
- No inappropriate dark/field presentation
- Route-aware, premium nav
- Loading/empty/error handled correctly
- Content-zone action anti-pattern removed

---

## Wave 02 — Reporting dashboard, queue, and inspections rebuild
### Objective
Convert the three most important operational surfaces into true workspaces.

### Included work
1. Reporting Period Dashboard redesign
2. Review Queue redesign
3. Inspections redesign
4. Safety-local list and row primitives
5. Responsive list/table/card behavior

### Exit criteria
- Dashboard has actual command value
- Queue and inspections behave like governed list pages
- Narrow layouts remain credible

---

## Wave 03 — Detail views, upload productization, and incidents decision
### Objective
Finish the workflow depth and remove release-credibility blockers.

### Included work
1. Upload workflow redesign
2. Inspection detail rebuild
3. Project-week detail rebuild
4. Incidents MVP or route removal
5. Provenance/metadata improvements

### Exit criteria
- Upload feels productized
- Detail pages are responsive and authoritative
- No placeholder routes remain in active nav

---

## Wave 04 — Evidence-backed closure
### Objective
Make flagship claims defensible.

### Included work
1. Expand Playwright coverage
2. Add hosted screenshot matrix
3. Add scorecard closure record
4. Confirm package result matches source intent

### Exit criteria
- Hosted behavior proven
- Route and state quality proven
- Breakpoint behavior proven
- Scorecard can be rerun credibly
