# Prioritized Enhancement Plan

## 1. Establish the Safety responsive contract
- **Gap closed:** G-02, G-10
- **Implementation direction:** define explicit Safety application modes and apply container-aware or slot-aware behavior where nesting materially changes quality
- **Expected impact:** major score lift in breakpoint, host resilience, and homepage integration
- **Cross-layer implications:** shell, page shells, local primitives, stylesheet, evidence workflow
- **When:** now
- **Type:** structural redesign

## 2. Rebuild the Upload surface into a flagship intake experience
- **Gap closed:** G-01, G-03, G-07
- **Implementation direction:** redesign intake around:
  - reporting period readiness
  - workbook selection and file summary
  - validation/progress runway
  - clear blocked-state handling
  - post-submit outcome pathways
- **Expected impact:** major product-identity improvement; strongest first-impression gain
- **Cross-layer implications:** page composition, upload controls, result banners, possible new local primitives
- **When:** now
- **Type:** structural redesign

## 3. Upgrade the authored Safety visual language
- **Gap closed:** G-01, G-06, G-08
- **Implementation direction:** strengthen the local surface family:
  - masthead authority
  - section hierarchy
  - action cluster language
  - provenance / authority framing
  - summary rails
- **Expected impact:** broad lift across composition, purpose-fit, and identity
- **Cross-layer implications:** shared page components and CSS
- **When:** now
- **Type:** structural redesign

## 4. Recompose the Reporting Period and Review surfaces around triage and action
- **Gap closed:** G-04, G-05, G-07
- **Implementation direction:** move beyond filter-bar + table as the dominant first-read pattern
- **Expected impact:** high lift in surface composition and interaction completeness
- **Cross-layer implications:** dashboard/list pages, table row affordances, summary bands
- **When:** now
- **Type:** structural redesign

## 5. Deepen the detail surfaces
- **Gap closed:** G-06, G-08
- **Implementation direction:** improve findings, section summaries, provenance, and source-workbook treatment with stronger side panels / grouped evidence framing
- **Expected impact:** medium-high lift in trust and decision speed
- **Cross-layer implications:** inspection detail, project-week detail, findings list, score strip
- **When:** next
- **Type:** refinement plus redesign

## 6. Close accessibility and hosted-proof gaps
- **Gap closed:** G-09, G-10
- **Implementation direction:** remove tactical affordance exceptions where possible and require hosted-evidence capture
- **Expected impact:** needed for acceptance closure
- **Cross-layer implications:** controls, test/evidence workflow, SharePoint-hosted validation
- **When:** next
- **Type:** refinement
