# Spotlight Gap Register

| Gap | Category | Current condition | Why it underperforms | Severity | Direction | Affected seams | Refinement vs redesign |
|---|---|---|---|---|---|---|---|
| Missing-media billboard | Media / hierarchy | Absent or failed images leave a large placeholder slab as the dominant object | The empty region becomes the story and buries actual project content | Critical | Add distinct missing-media posture with compressed media + title-led layout | `FeaturedMedia.tsx`, `project-spotlight-surface.module.css`, list source, normalization | Redesign |
| Weak first-view value in dominant slot | Composition | Title and summary start too low on large screens | The row’s major slot does not communicate the featured project quickly enough | Critical | Rebalance hero height, content overlay, and above-the-fold hierarchy | `layout-mode.ts`, `FeaturedSlot.tsx`, CSS | Redesign |
| Wide / medium default-open posture without enough vertical discipline | UX / responsive | Details and history can remain expansive while media already consumes major height | The page feels long before it feels useful | High | Tighten default-open policy or make it conditional on real headroom | `layout-mode.ts`, `HistoryDisclosure.tsx`, `FeaturedSlot.tsx` | Refinement |
| Compact / minimal still too image-led | Responsive UX | Smaller states collapse disclosures but still spend too much height on media context | Constrained screens need faster message delivery | High | Make small modes more selective, especially without real image content | `layout-mode.ts`, `FeaturedMedia.tsx`, CSS | Redesign |
| Featured CTA fragility | Task flow | If item CTA is thin or absent, the featured story can become a soft dead-end | Flagship modules need a clear next action | Medium | Add fallback CTA policy tied to section-level destination | normalization, `FeaturedSlot.tsx` | Refinement |
| History rail visually too heavy | Secondary content strategy | The rail is explicit and subordinate in concept, but still a large block when expanded | It competes with the featured story and adds height quickly | Medium | Shrink rail mass, support preview posture, or default-closed policy | `HistoryDisclosure.tsx`, `SupportingRail.tsx`, CSS | Refinement |
| Token discipline breach | Styling governance | Raw hex / rgba / px values dominate the surface CSS | Harder to govern, reuse, and prove benchmark compliance | High | Add governed local token seam and replace literals | CSS module, new token files / variants | Refinement |
| Primitive underuse | Premium-stack compliance | Separator, overlay, and panel behavior are still partially hand-rolled | Surface quality and maintainability trail the benchmark | Medium | Adopt stronger governed primitives where relevant | `index.tsx`, `TeamStrip.tsx`, related primitives | Refinement |
| Hosted proof gap | Closure | Stories and docs exist, but the hosted result is still visibly under target | “Documented” is not the same as “closed” | High | Add hosted visual proof, runtime checks, and no-regression acceptance criteria | stories, e2e, screenshots, closure docs | Refinement |
