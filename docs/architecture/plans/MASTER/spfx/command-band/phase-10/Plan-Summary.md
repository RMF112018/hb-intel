# Plan Summary

## Objective

Produce a stronger combined package for the Priority Actions / homepage launcher problem space by reconciling the two attached packages against the live `main` branch, binding doctrine, and external research.

## Bottom line

The prior packages were directionally right about the **render seam shift** and several surface-quality problems, but they were still too shallow in four places:

1. they did not push hard enough on **adapter and contract lossiness**
2. they did not fully clean up **authoring/governance drift** left behind by the launcher transition
3. they did not identify an internal **visible-count contradiction** across launcher layers
4. they did not isolate **label clarity, truncation rescue, link semantics, and hosted proof** into explicit closure units

## Package decision

This enhanced package keeps the valid prior findings, expands the issue set, and increases the prompt count from 7 to 10 so the work can close cleanly without hidden deferrals.

- preserve the strong data and host-fit seams
- rebuild weak launcher primitives rather than cosmetically tuning them
- restore semantic service identity through the adapter and icon mapping
- govern overflow as a real secondary launcher, not a dump list
- reconcile contradictory breakpoint/count rules
- improve accessibility and label clarity without reducing density
- prove final closure through tests, inspectable runtime markers, and hosted cutover checks
