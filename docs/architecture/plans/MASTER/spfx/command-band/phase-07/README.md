# Priority Actions Rail — Enhanced Combined Audit and Remediation Package

## Purpose

This package supersedes the attached audit/remediation pair with one combined package that is more explicit, more repo-truth-bound, and more forceful for local code-agent execution.

It is built around a simple repo-truth conclusion:

- the **homepage ownership model is already largely correct**
- the **shared flagship rail is not yet productized enough**
- the **existing package pair is directionally useful but under-bounded**
- the remaining work is **not** a light styling pass

## Bottom line

The current live implementation already has meaningful strengths:

- wrapper-owned pre-shell placement
- explicit `homepage-flagship` context
- list-source vs manifest-fallback resilience
- container-aware device/presentation seams
- multiple overflow strategies
- existing regression seams

The package pair you attached correctly recognized that the live surface is still too list-first and underpowered for a flagship homepage band.

However, the attached pair does **not** push hard enough on the real repo seams that now matter most:

- the shared flagship/default contract split
- the application-level breakpoint seam in `priorityActionsPresentation.ts`
- the already-authored entry-stack gutter and width-governance seam
- the need to lock behavior with stronger tests, not just redesign language
- the need to prove hosted SharePoint closure with explicit evidence

## What this package changes

Compared with the attached pair, this package:

1. separates **architecture that should be preserved** from **surface grammar that must be redesigned**
2. downgrades screenshot-led conclusions and upgrades code-seam evidence
3. adds explicit treatment of:
   - `priorityActionsPresentation.ts`
   - `HbHomepageEntryStack.module.css`
   - `hbHomepageWrapperConfig.ts`
