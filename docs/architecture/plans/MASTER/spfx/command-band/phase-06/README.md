## File naming convention

- `AUDIT-*` files are repo-truth audit / analysis artifacts.
- `PROMPT-*` files are local code-agent execution prompts.
- `README.md` and `Plan-Summary.md` are package-level guide files.

This package executes the attached objective: perform a narrowed, granularly exhaustive package-vs-repo reconciliation audit of the existing `PriorityActionsRail` / `hbHomepage` audit and prompt packages, then replace them with one stronger combined package for local code-agent execution.

## Net judgment

The live repo supports the **core architectural thesis** from the earlier package:

- `hbHomepage` owns a wrapper-level, pre-shell actions region.
- `PriorityActionsRail` is embedded there as a React surface.
- The rail is **not** a shell occupant, slot resident, or shell band member.
- The shell begins after the rail.

That architecture should be preserved.

The hosted result is still below bar because the current implementation expresses the rail as a **nested bordered card** inside a wrapper envelope that mirrors shell width and padding too closely. The result reads as a cautious utility card instead of a flagship homepage command band.

## What this package changes relative to the attached packages

This package is intentionally stronger in five ways:

1. It ties findings to **specific live repo seams and actual current code** rather than relying on broad judgment alone.
2. It decomposes the problem into **separate executable remediation units** instead of a small number of large prompts.
3. It adds **research-backed implementation guidance** for overflow, container-aware layout, and accessibility.
4. It defines a much tighter **validation and closure standard**, including build, test, authoring-mode, hosted SharePoint, zoom, and constrained-state proof.
5. It adds a dedicated **documentation/closure refresh step** so closure language cannot drift ahead of actual proof.

## Core decision lock

### Preserve
- wrapper-owned pre-shell rail placement
- separation between wrapper integration and shell occupant semantics
- standalone rail mountability outside `hbHomepage`
- container-aware shell logic and first-lane governance
- breakpoint-governed visible-action budgeting

### Change now
- the public rail surface contract and visual authority
- the wrapper envelope and hero-to-actions transition
- overflow interaction quality and accessibility hardening
- regression resistance around presentation / breakpoints / tests
- documentation and hosted closure proof
