# Priority Actions Rail — Wave 01 Enhanced Repo-Truth Package

## Objective

This package replaces the attached `wave-01` implementation package with a tighter, deeper, repo-truth-grounded execution package for the `PriorityActionsRail` and `PriorityActionsRailAdmin` workstream.

The attached package identified three real gap clusters, but it materially under-scoped the actual work still required in `main`.

This package is built to help a local code agent execute the remaining work cleanly, prove closure honestly, and avoid false completion.

## What changed relative to the attached wave package

The original package:
- correctly identified the two largest admin/product gaps and the shared-surface gap
- under-described the maturity already present in the repo
- under-described the actual remaining closure work
- omitted multiple now-in-scope workstreams required for truthful end-state closure

This enhanced package adds missing prompt coverage for:
- contract / validation / write-integrity hardening
- public-runtime and preview alignment with the authored config model
- token-discipline and visual-system cleanup
- hosted validation and closure-proof refresh

## Why the prompt count increased

The attached package assumed three large prompts were enough.

Repo truth says otherwise.

The remaining work now spans at least six distinct closure units:
- authoring-state correctness
- contract and validation hardening
- shared-surface rebuild
- consumer/runtime integration
- maintainer-product completion
- final doctrine / proof closure

Keeping those concerns collapsed would increase drift, hide dependencies, and make false closure more likely.

## Run order

Run the prompts in order.

The sequence is dependency-aware:
1. fix identity and lifecycle correctness first
2. harden contracts and validation before UI polish relies on them
3. rebuild the shared surface family before consumer alignment
4. align public runtime and preview with the real config model
5. finish the admin product on top of the corrected state and surface model
6. remove token drift and complete honest closure proof

## Non-deferral posture

This package is not written as a “good enough for now” bundle.

If a gap materially affects:
- correctness
- product quality
- trust in maintainer workflows
- doctrine alignment
- responsive behavior
- accessibility
- proof of closure

and it belongs to this wave, it is included here and must be addressed now.
