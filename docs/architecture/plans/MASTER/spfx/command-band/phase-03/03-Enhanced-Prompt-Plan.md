## Why the new package is materially better

### Better repo-truth grounding
It recognizes that the repo already has strong seam work and focuses the agent on the remaining closures rather than rebuilding solved layers.

### Better decomposition
It separates:
- correctness
- contract rigor
- shared primitive rebuild
- consumer alignment
- maintainer product completion
- proof and documentation closure

### Better sequence
It orders the work so each later prompt can assume stronger foundations.

### Better closure protection
It adds explicit proof requirements for:
- reorder correctness
- validation breadth
- preview fidelity
- permission states
- hosted evidence
- docs refresh

### Better doctrine alignment
It forces token cleanup, responsive honesty, accessibility-safe overflow patterns, and public/admin parity.

### Better protection against cosmetic fixes
Each prompt is scoped to a bounded closure unit with acceptance evidence, not just broad “improve this” language.
