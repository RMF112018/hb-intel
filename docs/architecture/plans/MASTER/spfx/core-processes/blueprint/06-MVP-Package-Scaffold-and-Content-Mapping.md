Build a scaffold broad enough to support all four primary corridors at launch, while providing the deepest first refinement in the turnover corridor.

## Content mapping strategy from current library

The current procedures-manual library should be mapped in three passes.

### Pass 1: Corridor mapping
Map current folders/files to parent corridor packages:

- Construction Operations startup/turnover content
- buyout/procurement content
- construction-management controls content
- financial reporting support
- superintendent support materials
- Safety
- Legal
- Quality Control

### Pass 2: Child-package extraction
Identify discrete standards or modules that deserve individual package treatment.

Criteria:

- high role specificity
- frequent use
- high handoff importance
- strong operational consequence if misunderstood
- likely direct search target

### Pass 3: Source reference classification
For each relevant source item:

- classify by content type
- designate canonical vs derivative
- assign package relationships
- assign role relevance
- assign lifecycle relevance
- define preview/open behavior
- define trust/freshness posture

## Package assembly rule

Do not build packages as document collections alone.

Every package must contain:

- in-app operating brief
- governed metadata
- typed source references
- package hierarchy and relationships
- trust/freshness behavior
- role/lifecycle context

## Near-parity scaffold rule

The public shell, companion, and content model must be able to represent all four primary corridors at launch.
“Near parity” does not require equal polish, but it does require:

- visible corridor presence
- package hierarchy
- role mapping
- lifecycle mapping
- source references
- trust markers
- search inclusion
- companion governance support

## Deep-implementation rule for A

Corridor A should receive the most refined first pass in:

- package quality
- role overlays
- handoff clarity
- source curation
- supporting-package integration
- public operating-path quality
- search quality
- feedback and trust instrumentation
