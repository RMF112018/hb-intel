# Prompt-06 — Phase 9: Final Release Closure and Signoff Report

## Objective

Produce the final Phase 9 release closure report for the Accounting app and connected Project Setup / provisioning workflow.

## Instructions

You are working inside the live repository.

Do not re-read files that are still within your current context or memory.

This prompt is the final consolidation step for the entire program of work. Use repo truth and the outputs from the earlier Phase 9 prompts to produce an evidence-based closure and signoff report.

## Deliverables

Create or update:
- `docs/architecture/reviews/phase-9-final-release-closure-and-signoff-report.md`

The report must include:
1. Executive Summary
2. Scope Closed
3. Evidence Reviewed
4. Readiness by Category
   - code readiness
   - deployment readiness
   - pilot readiness
   - cutover readiness
   - operational readiness
5. Remaining Risks
6. Remaining External Dependencies
7. Explicit Go / No-Go Recommendation
8. Recommended Immediate Next Actions
9. Closure Statement

## Required behavior

- Be explicit about what is fully complete, partially complete, or externally blocked.
- Do not overstate readiness.
- Distinguish between “ready to pilot” and “ready for production cutover” if the evidence requires it.
- Where uncertainty remains, state it plainly.

## Acceptance criteria

- The final report is evidence-based and auditable.
- Readiness language is precise.
- The report can support an actual release decision.
