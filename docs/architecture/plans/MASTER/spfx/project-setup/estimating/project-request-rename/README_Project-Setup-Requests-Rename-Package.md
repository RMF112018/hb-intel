# Project Setup Requests Rename — Prompt Package

This package contains a sequenced set of prompts for a local code agent to rename the current SPFx app from **Estimating** to **Project Setup Requests** and update package identity, documentation, architecture maps, and release/readiness artifacts.

## Recommended execution order

1. `Prompt-01_Repo-Truth-Audit-and-Rename-Plan.md`
2. `Prompt-02_Implement-App-Package-Rename.md`
3. `Prompt-03_Update-Docs-and-Architecture-Separation.md`
4. `Prompt-04_Final-Verification-and-Handoff.md`

## Optional alternative

- `Prompt-05_Consolidated-Rename-Execution.md`

Use the consolidated prompt only if you want the agent to perform the full effort in one pass. The staged sequence is preferred for control, traceability, and cleaner verification.
