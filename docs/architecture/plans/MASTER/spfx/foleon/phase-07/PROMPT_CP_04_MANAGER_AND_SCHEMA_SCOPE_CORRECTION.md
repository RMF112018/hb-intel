# CP-04 — Manager and Schema Scope Correction

## Objective

Correct the future schema/Manager plan so the Foleon Manager remains a placement/readiness/control surface, not an article authoring or content management system.

## Correct model

Foleon Manager should help HB Intel content managers:
- monitor available Foleon publications;
- select the active Company Pulse edition;
- verify embed/readiness state;
- manage placement and display metadata;
- manage archive/display windows;
- diagnose readiness issues.

Foleon Manager should not:
- author Company Pulse stories;
- manage event/recognition/operations article items;
- duplicate the Foleon publication's structure;
- become a CMS competing with Foleon.

## Metadata that may be appropriate

- active edition;
- title;
- teaser/summary override only if explicitly approved;
- hero/thumbnail URL;
- display window;
- archive group;
- audience/cadence;
- readiness state;
- source Foleon URL;
- embed URL;
- publish status.

## Metadata that should remain in Foleon

- article/story body;
- layout;
- authoring;
- rich media placement;
- video sequencing;
- category sections;
- full publication navigation;
- recognition lists;
- event details;
- operations articles.

## Output

Generate a decision note and update any implementation plans/prompts that wrongly imply HB should build a multi-article Company Pulse feed.

No code changes unless explicitly approved.
