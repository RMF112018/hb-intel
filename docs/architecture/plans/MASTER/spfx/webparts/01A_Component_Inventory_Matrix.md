## Ownership And Maintenance Rules

- Every component has an explicit non-developer content owner and a bounded configuration surface in property pane or list-backed content.
- Engineering ownership in this phase is limited to integration seams, shared primitives, and guardrails; content lifecycle remains with site owners.
- No component is permitted to depend on manual code edits for routine rotations, swaps, or copy refreshes.

## Prompt-01 Closure Check

- All ten first-release webparts are explicitly assigned a lane.
- All ten have an explicit data/content owner and maintenance expectation.
- No component remains in a `decide later` state.
