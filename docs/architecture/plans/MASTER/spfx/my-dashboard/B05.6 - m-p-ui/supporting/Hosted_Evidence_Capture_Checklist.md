# Hosted Evidence Capture Checklist  
## My Projects flagship validation

## Required screenshots

- [ ] Standard laptop — resting card
- [ ] Desktop — resting card
- [ ] Tablet landscape — resting card
- [ ] Tablet portrait — resting card
- [ ] Phone portrait — resting card
- [ ] Phone landscape / short-height
- [ ] Launch menu open on a tile near left side
- [ ] Launch menu open on a tile near right edge / constrained edge
- [ ] Portfolio browser open — desktop drawer
- [ ] Portfolio browser open — phone/full-sheet posture
- [ ] Search result state
- [ ] No-result state
- [ ] One degraded/partial state if practically available
- [ ] SharePoint edit/authoring posture if practical

## Runtime checks

- [ ] No console errors caused by menu/browser logic
- [ ] No clipped launch menu at normal desktop width
- [ ] No primary action hidden at common zoom
- [ ] Portfolio browser closes with Escape
- [ ] Focus returns to originating trigger after close
- [ ] Search input receives initial focus on browser open

## Evidence naming suggestion

Use a stable evidence naming pattern, for example:

```text
my-projects-desktop-resting.png
my-projects-standard-laptop-resting.png
my-projects-phone-portfolio-browser.png
my-projects-launch-menu-edge-case.png
```
