# Plan Summary

## Objective
Elevate the HB Homepage shell from a disciplined but still cautious modular shell into a width-confident, container-aware, doctrine-compliant homepage composition system.

## Core findings
1. Architecture exists and should be preserved.
2. Layout grammar is too limited.
3. Measurement truth is split.
4. Hosted-surface fit contracts are underdeveloped.
5. Width use is too timid.
6. Proof of closure is too weak.

## Required program
- unify entry-state authority,
- expand recipe-based shell layouts,
- formalize occupant shell-fit contracts,
- harden first-lane logic,
- align launcher governance,
- improve ultrawide density,
- and add black-box closure proof.

## Non-negotiable closure conditions
- no horizontal overflow for ordinary content,
- safe reflow at constrained widths / zoom,
- launcher and shell use the same entry-state truth,
- shell recipes are governed and validated,
- hosted occupants declare stable nested modes,
- and the homepage visibly uses space better than the current rendered state.
