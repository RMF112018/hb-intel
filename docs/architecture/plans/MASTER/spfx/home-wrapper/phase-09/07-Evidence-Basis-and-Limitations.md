# Evidence Basis and Limitations

## Evidence basis used for this package

### 1. Live repo seams
The package was derived from the live `main` branch seams governing:
- `apps/hb-homepage/`
- `apps/hb-webparts/src/webparts/hbHomepage/**`
- directly relevant shell, preset, breakpoint, launcher, registry, and validation files

### 2. Repo doctrine / benchmark material
The package cross-checked implementation direction against:
- `docs/reference/spfx-surfaces/governing-standard.md`
- `docs/reference/spfx-surfaces/benchmark/homepage-overlay/*.md`
- `docs/reference/spfx-surfaces/benchmark/README.md`
- related conformance / closure standards in the same benchmark area

### 3. Supplied screenshots
The rendered screenshots were used as evidence for:
- conservative width usage,
- centered/narrow visual posture,
- and mismatch between architectural potential and current output quality.

### 4. Public subject-matter research
The package used public guidance on:
- responsive design,
- container queries,
- component-level micro-layouts,
- and WCAG reflow expectations.

## Limitations

## Limitation 01 — Prior package zips were missing
The named prior packages were referenced in the prompt but not attached in this session.

**Impact:**  
No literal package-vs-package reconciliation was possible.

**Mitigation:**  
This package replaces the missing prior package pair with a stronger repo-truth deliverable.

## Limitation 02 — This package is an audit/remediation package, not an implementation diff
This package does not modify repo code directly.

**Impact:**  
Closure still depends on correct code-agent execution.

**Mitigation:**  
The prompts are intentionally explicit, forceful, and proof-oriented.

## Limitation 03 — Screenshot evidence captures visible symptoms, not all runtime states
The screenshots are strong evidence of current rendered weakness, but they do not show all host widths, zoom states, or fallback states.

**Impact:**  
Rendered closure must still be proven through the validation harness prompt.

## Confidence statement

Despite the missing prior zip packages, confidence is high in the core conclusions of this package because they are supported jointly by:
- live repo seams,
- doctrine requirements,
- rendered screenshots,
- and public best-practice research.
