# Recommended Implementation Waves

## Important note

These waves are **sequencing guidance only**.

They are **not** permission to defer the work.  
All listed work remains in scope for the current remediation program.

---

## Wave 1 — Core authority and shell grammar
Execute:
- Prompt 01 — unify entry-stack measurement and breakpoint truth
- Prompt 02 — expand shell layout grammar and band recipes
- Prompt 08 — harden preset governance and config bounds

### Deliverable outcome
A shell that has:
- one width truth,
- a richer but governed layout vocabulary,
- and schema/validation protection against configuration sprawl.

---

## Wave 2 — Hosted fit and first-lane behavior
Execute:
- Prompt 03 — add hosted-surface shell-fit contracts
- Prompt 04 — rebuild first-lane governance and vacancy handling
- Prompt 05 — align launcher band with shell governance

### Deliverable outcome
A shell that:
- knows what each hosted surface can safely do,
- promotes first-lane content deterministically,
- and keeps launcher behavior aligned with shell density and width truth.

---

## Wave 3 — Width confidence and closure evidence
Execute:
- Prompt 06 — improve ultrawide width use and shell density
- Prompt 07 — add black-box shell validation and proof harness

### Deliverable outcome
A rendered homepage that:
- uses desktop space more confidently,
- remains reflow-safe,
- and has objective proof of closure.

---

## Exit criteria for the full program

The full program exits only when all three wave outcomes are true together:
1. structure is stronger,
2. rendered composition is visibly stronger,
3. and proof of closure is stronger.
