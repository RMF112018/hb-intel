# 05 — Elite Homepage Composition Questions

These are the questions strong teams ask when reviewing world-class homepages and operating surfaces.

## Composition questions

1. What does the user understand in the first two seconds?
2. Is the first view saying “this is useful” or only “this is branded”?
3. Does the top of the page earn its height?
4. Is the first meaningful task surface visible before the user has to decide whether the page is worth scrolling?
5. Are the most important actions obvious without making everything equally loud?
6. Does the page reveal a clear narrative from top to bottom?
7. Are sections grouped by user intent, not by authorship convenience?
8. Is secondary content truly secondary, or just placed lower?
9. When content is empty, does the page become quieter in a disciplined way, or simply blank?
10. Can the layout degrade under constraint without looking broken or apologetic?

## Governance questions

1. Which layout decisions are product decisions and which are editor decisions?
2. If a maintainer moves things later, what prevents quality collapse?
3. Can the system reject bad placements before they ship?
4. Can the shell reason about content strength, emptiness, and slot comfort?
5. Does the flagship experience still depend on manual page assembly outside the governed system?

## Applied to HB Homepage

### Strong answers already present
- There is a real shell architecture.
- There are protected decisions.
- There is a meaningful preset model.
- There is a future control-panel boundary.

### Weak answers right now
- The first two seconds are still too brand-heavy relative to value.
- The page still depends on non-governed OOB utility treatment.
- Empty states still own too much composition.
- The governed shell is not yet fully dictating the flagship page outcome.

## What “elite” means here

It does **not** mean more animation, more cards, or more chrome.

It means:

- higher signal density
- clearer priority
- tighter orchestration
- less wasted height
- smarter empty-state behavior
- stronger governance over what can appear where
