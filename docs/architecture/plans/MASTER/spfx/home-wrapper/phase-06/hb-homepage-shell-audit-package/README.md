## Core finding

The current homepage is not failing because it lacks components. It is failing because the page-level composition is still behaving like several adjacent assemblies instead of one governed flagship entry experience. The largest issue is the entry stack:

1. a large signature hero
2. a separate OOB Quick Links directory row
3. a shell whose first meaningful lane arrives too late and can be pre-empted by empty-state modules

## Most important actions

1. Replace the OOB Quick Links row with the governed `PriorityActionsRail`.
2. Make the first shell lane non-empty-first and flagship-first.
3. Ensure the first meaningful shell lane begins on first load on the 14-inch MacBook Pro baseline.
4. Keep the shell's protected decisions and preset architecture, but reduce page-authoring dependence on mixed OOB + governed surfaces.
5. Add the control-panel groundwork now, but keep entry-stack rules code-governed.
