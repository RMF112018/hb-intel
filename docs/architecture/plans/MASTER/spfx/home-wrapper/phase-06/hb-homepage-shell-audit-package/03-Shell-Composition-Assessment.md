# 03 — Shell Composition Assessment

## Current composition diagnosis

The current page has the right ingredients, but the ordering and orchestration are still weak.

### What is working

- The hero has visual authority.
- The modular homepage surfaces have differentiated personalities.
- The shell architecture already recognizes bands, slots, pairing, stacking, and control boundaries.

### What is not working

#### 1. The top of the page is not value-dense enough
The first screen is over-invested in brand and utilities relative to immediate operational value.

#### 2. Utility is visually overexposed but semantically under-prioritized
The quick links row is reading as a broad directory, not a shaped set of primary actions.

#### 3. Empty states are getting premium placement
The screenshots make this especially clear. When there is little or no content in several surfaces, the shell still gives them height, sequence, and page weight. That makes the homepage feel empty before it feels useful.

#### 4. The page still feels authored as adjacent modules, not one flagship sequence
The separate hero + OOB quick links + shell stack still feels additive instead of orchestrated.

## Recommended hierarchy

### Dominant
- Signature hero
- first meaningful operational shell lane

### Secondary
- priority actions band
- project spotlight / operational awareness
- strongest current newsroom or leadership signal

### Tertiary
- people/culture
- recognition
- safety updates when there is no urgent signal
- archive or supporting rails

### Progressive disclosure
- lower-frequency tools
- overflow actions
- archive browsing
- secondary editorial rails
- companion or admin-only behaviors

## Recommended grouping

### Group 1 — Entry stack
- hero
- priority actions
- first shell lane

### Group 2 — Work acceleration
- optionally `ToolLauncherWorkHub`
- optionally `SmartSearchWayfinding`
- only if their container-fit and page-story role are explicit

### Group 3 — Communications
- Company Pulse
- Leadership Message

### Group 4 — Culture and recognition
- People & Culture Public
- HB Kudos
- keep them related, but do not overexpose both above stronger operational content

### Group 5 — Operational context
- Project Portfolio Spotlight
- Safety & Field Excellence

## Best current first-lane strategy

For the live homepage, the strongest first shell lane should be:

- Project Portfolio Spotlight as dominant
- one compatible supporting surface only if that secondary surface is non-empty and shell-fit-safe

If Company Pulse is empty or stale, it should not hold the paired secondary slot just because the preset says so.

## Key composition rule to add now

**The first shell lane must be non-empty-first, not preset-first.**

That means:

- evaluate candidate occupants for content availability and strength
- promote the best available pair or single dominant surface
- demote empty / invalid / stale-low-value surfaces below the early page narrative

This is the single most important shell-composition upgrade.
