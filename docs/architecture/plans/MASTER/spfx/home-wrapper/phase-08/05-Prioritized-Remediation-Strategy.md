# 05 — Prioritized Remediation Strategy

## Strategy objective
Close the host-fit problem without destabilizing the shell’s strong ownership model.

## Required sequence
### Step 1 — Establish one outer envelope authority
Do this first.

Until the code makes it obvious which seam owns the outer page-canvas fit contract, everything downstream stays ambiguous.

### Step 2 — Preserve deliberate inner-region styling inside that shared authority
Do this second.

The goal is not flat sameness. The goal is that the actions strip and shell body live inside the same outer contract while still being allowed to express different internal inset behavior.

### Step 3 — Rebase shell measurement on authoritative usable width
Do this third.

After the outer contract exists, rebase `useShellContainer` and any related measurement seam so runtime state derives from authoritative usable width rather than from self-referential shell geometry.

### Step 4 — Harden the actions-region containment boundary
Do this fourth.

Preserve wrapper ownership, but make the actions region a disciplined participant in the same page-canvas fit model.

### Step 5 — Strengthen diagnostics
Do this fifth.

A shell that cannot explain its own width truth is hard to validate and hard to maintain.

### Step 6 — Add focused proof
Do this sixth.

Keep tests narrow and meaningful. Do not explode the test suite into noisy snapshot churn.

### Step 7 — Produce hosted validation evidence and a closure report
Do this last.

Closure should fail if this evidence is missing or vague.

## What this strategy avoids
- broad redesign of the shell
- reclassification of the embedded rail as a shell occupant
- child-surface refactoring to hide shell-fit mistakes
- cosmetic clipping as fake closure

## Expected result
After this strategy, the codebase should have:
- one declared outer fit contract
- intentional region-specific inset policy
- truthful usable-width measurement
- stronger diagnostics
- meaningful automated proof
- hosted evidence that the screenshot-class defect is closed
