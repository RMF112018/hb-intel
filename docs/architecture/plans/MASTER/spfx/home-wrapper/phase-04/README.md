# Enhanced Wave 01 Homepage Shell Package

## Purpose

This package executes the shell-only objective defined in the attached prompt and replaces the existing Wave 01 package with a materially stronger audit and implementation set.

This package is **strictly shell-only**. It is not a module remediation package.

## What this package does better than the current Wave 01 package

- validates the current Wave 01 issue set against live repo truth
- distinguishes between work that is already partially implemented and work that is still missing
- sharpens the missing work around:
  - entry-stack policy and first-screen budgets
  - breakpoint rigor and short-height behavior
  - governed persistence and control-panel readiness
  - production orchestration seams across independently mounted top-band surfaces
  - harnesses, diagnostics, and closure proof
- gives your local code agent more operationally useful prompts with:
  - exact seams and symbols
  - explicit end states
  - stronger acceptance criteria
  - stronger guardrails
  - no-deferral language

## Core repo-truth judgment

The current Wave 01 package is **directionally correct but materially underpowered**.

The live repo already includes:
- shell entry-state typing and resolution
- container-aware shell sizing
- a preset library
- protected and configurable decision models
- Zod-backed validation
- slot-comfort and pairing logic
- automated tests for breakpoint policy, slot comfort, and shell validation

Because that work already exists, the new package focuses on **closing the real remaining shell gaps** instead of restating greenfield shell architecture work.

## Success standard

Wave 01 is complete only when all of the following are true:

- shell ownership boundaries are explicit and drift-resistant
- the entry stack has a shell-facing policy contract covering hero height, actions budget, first-lane visibility, and short-height handling
- breakpoint behavior is aligned to the governing spec and inspectable in diagnostics
- governed presets and persistence boundaries are explicit, validated, and normalization-safe
- production seams allow future unified entry-stack governance without ad hoc coupling
- a repeatable shell harness and closure matrix exist
- automated tests and proof artifacts confirm the shell across required device classes
