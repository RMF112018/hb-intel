# 05 — Recommended Implementation Waves

## Wave 01 — Flagship shell reconstruction

### Objective
Move `hbHomepage` from a tidy stacked orchestrator to a real flagship homepage shell.

### Scope
1. Realign the shared hero primitive to current homepage doctrine.
2. Build the flagship top-band into `hbHomepage`.
3. Introduce an explicit shell composition registry / layout contract.
4. Replace silent zone failures with visible premium degradation.
5. Clean up shell-level prop and zone contract drift.

### Why this is Wave 01
The homepage cannot become benchmark-grade until the shell itself has flagship authority. This wave establishes the top-band, shell governance, and degradation posture.

### Closure target
- homepage has a true flagship opening plane
- shell no longer reads as simple stacked zone order
- shell-level fallbacks and contracts are explicit
- no silent zone disappearance remains
