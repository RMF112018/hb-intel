# Shell Composition Assessment

## Current reading of the page
Today the shell reads as:
- a strong set of modules
- arranged in a fixed vertical sequence
- with limited shell-owned narrative logic

That is acceptable for a composed MVP.
It is not yet the strongest possible homepage composition.

## What currently works

### Company Pulse first is understandable
Opening the shell with Company Pulse gives the page an immediate newsroom / freshness signal.

### Leadership after Company Pulse creates tone contrast
Leadership Message acts as a calmer, more reflective pause after the more active newsroom module.

### Project Spotlight is a strong mid-page anchor
Project Portfolio Spotlight has enough visual authority and informational weight to operate as a flagship operational anchor.

### HB Kudos near the lower portion has revisit energy
Recognition content often benefits from lower-page persistence because it remains discoverable without displacing more urgent operational/editorial content.

## What is currently weak

### 1. The sequence is too equal-weight
Even though the modules are not visually identical, the shell gives them a largely equal sequential treatment:
- stack
- gap
- stack
- gap
- stack

That creates a compositional flattening effect.

### 2. Company Pulse and Project Spotlight compete
These are both dominant-story surfaces:
- each has a strong masthead
- each has a featured object
- each has a supporting rail
- each wants attention

When they are simply stacked in a linear sequence, they compete rather than collaborate.

### 3. Leadership Message is currently a section, not a lane strategy
Leadership is well-built as a surface, but the shell is not using it strategically.
It should be a **human/executive editorial counterweight**, not just the second item because that was the implementation order.

### 4. People & Culture is under-positioned and underpowered
People & Culture currently sits as a standalone lower-middle section, but its surface maturity does not justify equal-weight presentation next to the strongest modules.

### 5. The shell lacks explicit primary / secondary / tertiary logic
A flagship homepage should communicate:
- what matters most right now
- what supports that priority
- what can wait
- what remains discoverable without full prominence

The current shell does not yet do that at the shell level.

## Recommended organizational model

## Recommended dominant surfaces
### Primary dominant band
- `ProjectPortfolioSpotlight`
- `CompanyPulse`

These are the best candidates for primary dominance because they establish:
- operational confidence
- current-state awareness
- visible freshness
- revisit value

But they should not simply stack as equal modules.
They need shell-owned orchestration.

## Recommended supporting surfaces
### Secondary / contextual band
- `LeadershipMessage`
- `PeopleCulturePublic`

These should function as a people/context lane:
- executive framing
- culture / community / internal context
- less urgent than operational + newsroom content
- better suited to pairing or clustered treatment

## Recommended persistent recognition surface
### Recognition band
- `HbKudos`

HB Kudos deserves persistence because:
- it has strong engagement mechanics
- it supports repeat visitation
- it adds warmth and human energy

But it should not visually overpower the page’s operational/editorial core.

## Recommended future operational lane
### Safety / field intelligence band
- `SafetyFieldExcellence`

When introduced into the shell, this should not be treated as just another equal module.
It likely deserves either:
- co-equal operational pairing with Project Spotlight at wide widths, or
- a clearly defined secondary operational lane with recency emphasis

## Recommended grouping logic

### Wide viewport
Recommended shell grouping:
1. **Primary anchor lane**
   - Project Portfolio Spotlight
   - Company Pulse
   - governed as co-dominant anchors with clear precedence rules

2. **People + leadership lane**
   - Leadership Message
   - People & Culture Public
   - paired or sequential within a calmer lane

3. **Recognition lane**
   - HB Kudos
   - persistent, engagement-driven, lower than anchor lane

4. **Future operational / safety lane**
   - Safety Field Excellence
   - introduced with explicit operational priority logic

### Medium viewport
- keep one dominant anchor visible first
- demote paired lanes into sequenced sections
- preserve strong section headers and rhythm
- avoid simply shrinking everything equally

### Narrow / compressed viewport
- one clear dominant section at a time
- progressively disclose supporting depth
- demote dense supporting content into shorter summaries, expandable rails, or “view more” pathways

## What should be dominant, secondary, tertiary, or contextual

### Dominant
- Project Portfolio Spotlight
- Company Pulse

### Secondary
- Leadership Message
- HB Kudos

### Tertiary / contextual
- People & Culture Public in current form
- future optional modules not yet built

### Context-driven / progressive disclosure candidates
- archive rails
- secondary project lists
- longer supporting culture rows
- some recognition archive/feed depth
- future “more modules” drawer or governed rotator content

## What should not be fully configurable later
A future maintainer control panel should **not** be allowed to:
- break the flagship top-band hierarchy
- place too many dominant surfaces adjacent without compatibility rules
- make every module full-width
- freely destroy primary / secondary / tertiary structure
- create a parking-lot grid

## Composition verdict
The homepage does not have a module-quality problem first.
It has a **shell hierarchy problem first**.

The most important redesign move is to replace:
- static sequential stacking

with:
- authored, governed shell hierarchy
- explicit lane roles
- compatibility-aware module placement
- breakpoint-specific composition behavior
