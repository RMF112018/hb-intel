# Plan Summary — Wave 02

## Goal
Refine the modules and presets so the new shell behaves like a flagship homepage rather than a generic configurable container.

## Boundaries
This wave should not:
- weaken the governance introduced in Wave 01
- introduce freeform maintainer control
- treat every module as equal-weight
- re-read files already in active context unless needed to confirm drift, dependencies, or uncertainty after changes

## Closure standard
Wave 02 is only complete when:
- People & Culture is aligned with the homepage quality bar
- shell presets encode real hierarchy instead of neutral stacking
- future module insertion is planned via registry metadata
- cross-module conflicts are reduced across viewport sizes
