# Shell Composition Assessment

## Objective judgment

The shell now has the **technical means** to produce a premium homepage composition.
It does not yet use those means aggressively enough.

The main issue is no longer “the shell cannot compose.”
The main issue is “the current shipped composition still behaves too politely.”

## 1. Best organization of the applications

### Recommended dominant hierarchy
The homepage should prioritize by **operational value first**, then editorial value, then human/context value, then recognition.

Recommended order:

1. **Project Portfolio Spotlight** — primary operational anchor
2. **Company Pulse** — timely newsroom/supporting editorial anchor
3. **Leadership Message** — executive context band
4. **Safety Field Excellence** — high-value operational lane once active
5. **People & Culture Public** — human/culture lane
6. **HB Kudos** — persistent recognition lane

### Why this is better
- It puts project/work relevance near the top
- It still preserves timely company context
- It stops leadership and culture from crowding the operational opening
- It treats recognition as important but not structurally dominant

## 2. What should be dominant, secondary, tertiary, or contextual

### Dominant
- Project Portfolio Spotlight
- Company Pulse

### Secondary
- Leadership Message
- Safety Field Excellence

### Tertiary / contextual
- People & Culture Public
- HB Kudos

### Important clarification
“Tertiary” here does **not** mean unimportant.
It means these modules should not dominate the first decisive screen in the same way operational and timely editorial modules do.

## 3. What currently works

### Strong current strengths
- The shell no longer assumes every module is a dumb full-width card
- Pairing logic and preset logic now exist
- Company Pulse, Leadership Message, and Project Spotlight are strong enough to participate in real orchestration
- HB Kudos is robust enough to hold a later persistent lane without feeling disposable

## 4. What currently creates friction

### A. The default preset is too historical
It still reflects “one module after another” rather than “what should the homepage do first?”

### B. Project Spotlight is compositionally under-promoted
It is an operational anchor but is not currently treated as first-lane eligible in the occupant registry.

That is the single clearest composition-policy mismatch in the current shell.

### C. Leadership is too close to the front by default
Leadership Message is well built, but putting it ahead of Project Portfolio Spotlight weakens the homepage’s work-first identity.

### D. Recognition is correctly separated, but still lacks a ceiling contract in layout behavior
The code says recognition is contextual and not allowed to overpower the page, but the shell does not yet fully exploit that in preset grammar.

## 5. Recommended grouping / clustering / sequencing

### Recommended first shell lane
- **major:** Project Portfolio Spotlight
- **minor:** Company Pulse

This gives the homepage:
- operational visibility
- newsroom freshness
- immediate proof of value

### Recommended second band
- Leadership Message

This can remain full-width at first, or later pair with Safety once Safety is activated and tested.

### Recommended third band
- Safety Field Excellence once live

Safety should not be buried.
It should sit above human/culture and recognition once the shell activates it.

### Recommended fourth band
- People & Culture Public

This is where the human/culture lane belongs today.

### Recommended fifth band
- HB Kudos

Keep it revisit-worthy and persistent, but do not let it define the entire homepage hierarchy.

## 6. What should be progressive disclosure rather than always fully exposed

### Priority Actions
Already moving in the right direction through overflow.
Keep going.
Do not let the actions band become a flat directory again.

### HB Kudos archive / extended feeds
These are good candidates for progressive disclosure and already behave that way.

### Supporting editorial / secondary stories
The shell should favor:
- one strong first-lane pair
- then disciplined supporting lanes
- rather than exposing everything at equal scale

## 7. Which applications compete destructively today

### Current destructive or near-destructive conflicts
- **Leadership Message vs Project Portfolio Spotlight** in sequence
- **People & Culture Public vs benchmark-strength surfaces** in craft quality
- **default single-column stack vs the shell’s new orchestration capability**

The issue is less “visual clutter” and more “insufficient hierarchy differentiation.”

## 8. Compatibility with future tenant-admin governance

### Compatible now
- presets
- slot overrides
- occupant registry
- protected vs configurable decisions

### Still incompatible or underdeveloped
- no persisted configuration seam
- no strong footprint library
- no preview/validation authoring experience
- no explicit first-lane role contract for Project Spotlight
- no full shell grammar for optional occupants like Safety

## 9. Net composition verdict

The homepage shell now has the infrastructure to move beyond a stacked enterprise page.

The next step is to **change what the shell chooses to do with that infrastructure**:
- stronger first-lane composition
- better operational prioritization
- clearer contextual demotion of human/recognition lanes
- removal of the last major outlier in surface discipline
