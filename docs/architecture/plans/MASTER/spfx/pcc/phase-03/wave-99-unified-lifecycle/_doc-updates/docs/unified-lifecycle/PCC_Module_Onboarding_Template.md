# PCC Module Onboarding Template

## Purpose

Every future PCC module must complete this template before implementation.

## Required Sections

1. Official module name.
2. Primary surface.
3. Primary work center.
4. Secondary affinities.
5. Readiness signal behavior.
6. Priority Action behavior.
7. Project Memory contribution.
8. Traceability edges produced.
9. Traceability edges consumed.
10. Source-owned records referenced.
11. PCC-native records owned.
12. Security classes.
13. Redaction rules.
14. HBI eligibility.
15. Approval/checkpoint relationship.
16. Audit events.
17. Preview state.
18. Live integration gates.
19. Test requirements.
20. Forbidden behaviors.

## Mandatory Acceptance

A module is not implementation-ready until this template is completed with closed decisions and cross-referenced to the source-of-record matrix, route taxonomy, permission/redaction algorithm, and test gates.

## Reference JSON

Use `reference/module_onboarding_template.json` as machine-readable source.
