# Controlling Objective — PCC Unified Lifecycle Implementation

PCC Unified Lifecycle is the connective project operating layer across the Project Control Center. It links lifecycle events, memory records, role/stage lenses, related records, warranty traces, cross-project knowledge, and Ask-HBI grounded search while preserving source-of-record discipline.

## Required Architecture Position

- PCC remains one unified project operating layer.
- Unified lifecycle is not a shell route or workspace.
- Project Home and Project Readiness are the primary MVP presentation surfaces.
- Project Memory, Traceability, Warranty Trace, Cross-Project Knowledge, and Unified Search are governed read-model / component capabilities, not separate department apps.
- HBI answers must be source-lineage-grounded, cited, permission-filtered, and refusal-capable.
- Source systems remain systems of record.
- Live integrations remain future-gated.

## Required Exact Phrase

```text
Do not re-read files that are still within your current context or memory unless you need to verify stale, missing, or contradictory repo truth.
```
