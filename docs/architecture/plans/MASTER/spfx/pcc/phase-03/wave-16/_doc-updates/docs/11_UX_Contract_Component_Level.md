# 11 — UX Contract at Screen / Component Level

## Primary screens
1. Settings Home
2. Category Landing
3. Settings Table
4. Setting Detail Drawer
5. Change Request Drawer
6. Approval Handoff Panel
7. Audit History
8. Validation and Health
9. Role Visibility Matrix
10. Feature / Module Flags
11. External Systems Configuration
12. Security / Secret References
13. HBI Settings
14. Responsive Mobile View

## Required table columns
Setting, Category, Effective Value, Effective Source, Owner, Validation, Approval, Last Updated, Actions.

## Required drawer sections
Header, effective value, ownership, validation, edit policy, dependencies, audit preview, HBI explanation, footer actions.

## Accessibility
Keyboard focus returns to trigger after drawer/dialog close. Modal dialogs trap focus. Do not nest dialogs. Inline errors appear close to affected fields. Color is not the only status cue. Tables must have accessible headers.
