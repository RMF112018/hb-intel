# 04 — Prompt Sequence Overview

## Prompt 01 — Repo Truth and Doc Gap Audit

Read-only. Confirms current local state, Wave 16 docs, package baseline, and exact files to update.

## Prompt 02 — Runtime DTO / Read-Model Contract

Adds or updates documentation/package reference material for the Wave 16 runtime DTO contract and schema-to-model mapping.

## Prompt 03 — Fixture Scenario and Data Mapping Matrix

Defines deterministic fixture scenarios, known/unknown/degraded project data, redaction examples, and schema-to-fixture coverage.

## Prompt 04 — Role / Action / Redaction / Disabled Reason Matrix

Defines who can see/request/edit/recheck/escalate, redaction behavior, and disabled reason copy.

## Prompt 05 — Effective Value Resolution and Change Request Lifecycle

Defines pure setting resolution logic, override behavior, lifecycle statuses, future command payloads, and command-gated UX posture.

## Prompt 06 — Wave 14, Priority Actions, and Cross-Surface Handoff

Defines approval/checkpoint handoff, Priority Actions generation, and cross-surface source-lineage references.

## Prompt 07 — UI Component / State / Copy / Accessibility Contract

Defines screen decomposition, state ownership, mobile behavior, copy catalog, and accessibility/test hooks.

## Prompt 08 — HBI / Audit / Security / Test / Acceptance Closeout

Defines HBI allowed/refused behavior, business audit vocabulary, secret-reference display, full test matrix, acceptance criteria, and closeout.

## Prompt 09 — Fresh Reviewer Prompt

Audits the resulting gap-closed docs/package for developer readiness.
