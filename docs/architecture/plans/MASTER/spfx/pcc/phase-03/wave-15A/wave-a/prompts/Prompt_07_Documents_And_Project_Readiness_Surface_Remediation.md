## Role

You are the Documents and Project Readiness surface remediation agent.

## Objective

Remediate Documents and Project Readiness using completed shared patterns and first-impression lessons.

## Scope

## Non-Scope

No Project Home/Team & Access redesign except regression fixes. No Site Health, Settings, Approvals, External Systems work except shared dependencies. No backend/API work.

## Required Repo-Truth Inspection

Inspect Documents files, document control view model, source-state messaging, Project Readiness files/adapters, tests, screenshots, and Prompt 02-06 closeouts.

Clarify formal Project Record vs working-file lanes, review/permission states, readiness blockers, owner/next action hierarchy, lifecycle continuity, and source confidence.

## Required Tests

Run typecheck, Documents tests, Project Readiness tests, affected shared tests, and build if feasible. Add tests for Project Record distinction and readiness hierarchy where appropriate.

## Required Screenshot / Evidence Output

Capture before/after screenshots for Documents and Project Readiness across required widths and key states.

## Scorecard Impact

Targets Documents-specific clarity, readiness hierarchy, surface composition, state language, layout hierarchy, and product confidence.

## Closeout Requirements

Create Prompt 07 closeout with files changed, screenshots, command results, scorecard impact, and residual issues.

## Stop Conditions

Stop if document lane semantics conflict with approved document control architecture, or if readiness data requires backend changes.

## Standing Instruction

Do not re-read files still within your current context unless exact wording, line references, or changed repo state must be verified.

Begin with repo-truth inspection. Avoid unrelated changes. Avoid feature creep. Prefer shared primitives over one-off local styling. Update closeout documentation. Report exact files changed, commands run, validation results, residual issues, and any stop conditions.
