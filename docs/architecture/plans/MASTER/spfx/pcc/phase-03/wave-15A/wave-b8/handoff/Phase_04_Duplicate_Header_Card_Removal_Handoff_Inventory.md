This document inventories the duplicate / header-like bento cards
across all eight current PCC MVP surfaces and maps their content,
test, and Playwright/evidence impacts. It is **documentation-only** —
it does not authorize runtime removal, demotion, reordering, or
modification of any card. A subsequent local-agent prompt package
(Phase 04) consumes this inventory and executes removal in a
controlled tranche-by-tranche sequence. Until that package runs and
its own Hard Stops are satisfied, every duplicate-classified card
remains in production.

## 3. Phase Boundary

- Phase 03 (Wave B8) enriches conditional command-header
  metadata/content. It does **not** remove any duplicate bento card.
- Phase 04 removes or demotes duplicate bento header cards **only
  after** the shell command header has absorbed equivalent content,
  or the operational content has an explicit alternative home.
- Card-level `data-pcc-active-surface-panel` is currently a deprecated
  compatibility marker (Phase 2 closeout §10). Tests, e2e selectors,
  and committed evidence rely on its presence as one direct
  bento-child compatibility card per active surface.
- Broad `[data-pcc-active-surface-panel]` count in shell-rendered
  trees is intentionally `2` today: shell `<main role="tabpanel">`
  semantic owner + one direct bento-child compatibility card. Phase
  04 lowers that count to `1` in lockstep with test / e2e / evidence
  updates; nothing earlier may assume the post-removal count.
- Runtime removal of any card marker must update tests, Playwright
  surface registry/spec/evidence-writer, and committed live evidence
  in lockstep. No Phase 04 prompt is allowed to land a runtime
  change without the corresponding test/e2e/evidence update.
