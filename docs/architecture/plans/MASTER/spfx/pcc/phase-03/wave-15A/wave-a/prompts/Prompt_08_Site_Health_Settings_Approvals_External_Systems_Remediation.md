# Prompt 08 — Site Health, Settings, Approvals, and External Systems Remediation

## Role

You are the governance, risk, and integration surface remediation agent.

## Objective

Remediate Site Health, Control Center Settings, Approvals, and External Systems using completed shared patterns.

## Scope

Site Health components, Control Center Settings surface, Approvals surface, External Systems surface/cards/tests, and related copy/state/layout code.

## Non-Scope

No backend/API enablement. Do not add unapproved integrations. Do not alter first-impression surfaces except regression fixes.

## Required Repo-Truth Inspection

Inspect Site Health, Control Center Settings, Approvals, External Systems, source mapping/integration docs, tests, screenshots, and Prompt 02-07 closeouts.

## Required Implementation Work

Strengthen Site Health security/repair hierarchy, Settings governance ownership/locked/editable/preview distinctions, Approvals preview-safe queue/checkpoint content, and External Systems workflow value/mapping status.

## Required Tests

Run typecheck, affected surface tests, shared layout/state tests, and build if feasible. Add tests for governance ownership, preview-safe approvals, external system mapping status, and site health priority hierarchy.

## Required Screenshot / Evidence Output

Capture before/after screenshots for all four surfaces across required widths and key states.

## Scorecard Impact

Targets governance/risk/integration surface quality, state language, command-center hierarchy, interaction affordance, and product confidence.

## Closeout Requirements

Create Prompt 08 closeout with files changed, screenshots, tests, scorecard impact, and residual issues.

## Stop Conditions

Stop if remediation would require real integration enablement, credentials, Graph/Procore/Sage API changes, or a product decision about governance authority.

## Standing Instruction

Do not re-read files still within your current context unless exact wording, line references, or changed repo state must be verified.

Begin with repo-truth inspection. Avoid unrelated changes. Avoid feature creep. Prefer shared primitives over one-off local styling. Update closeout documentation. Report exact files changed, commands run, validation results, residual issues, and any stop conditions.
