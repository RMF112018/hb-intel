# Scorecard Review Checklist — Project Home

## P1 — PCC Product Strategy and Command-Center Clarity

- [ ] The page is clearly a command center, not a dashboard dump.
- [ ] The first scan tells the user what matters today.
- [ ] Project context and operating posture are visible together.

## P2 — Mold Breaker Differentiation

- [ ] Lifecycle continuity is visible.
- [ ] HBI is visible and grounded.
- [ ] The page shows how PCC unifies project records and source systems.

## P3 — Shell / Navigation / Project Context

- [ ] Active tab and active panel remain consistent.
- [ ] Project context is not duplicated confusingly.
- [ ] Project Home is the obvious starting surface.

## P4 — Layout / Bento / Hierarchy / Density

- [ ] Tier 1, Tier 2, and Tier 3 order is coherent.
- [ ] Dense content is progressively disclosed.
- [ ] Priority Actions no longer dominates the mobile page.

## P5 — Workflow / Interaction / Next Action

- [ ] Top actions identify owner, priority, due date, and source.
- [ ] Disabled/inert actions explain why and where action happens.
- [ ] No false execution affordances.

## P6 — State / Source / Read-Only

- [ ] Source boundaries are explicit.
- [ ] HBI is advisory only.
- [ ] Preview/demo/mock posture is explicit where applicable.

## P7 — Responsive / Field / Host Fit

- [ ] Phone and tablet first scan are usable.
- [ ] No horizontal overflow.
- [ ] Touch targets are safe.

## P8 — Accessibility

- [ ] Axe contrast findings resolved.
- [ ] State is not color-only.
- [ ] Keyboard and touch behavior are safe.

## P9 — Evidence / Validation

- [ ] Before/after evidence path is documented.
- [ ] Screenshot evidence captures below-fold state.
- [ ] Playwright evidence is treated as support, not final score.
