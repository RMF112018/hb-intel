## 7. High-Risk Enhancements

Patterns that could accidentally violate Phase 08 guardrails. Treat each as a non-negotiable veto during execution review.

- Sidebar reintroduction (plan §3.1) — prohibited.
- Active-panel marker regression — `data-pcc-active-surface-panel` must remain shell-owned on `main[role="tabpanel"]` (plan §3.2).
- Bento wrapper drift — no wrappers between `PccBentoGrid` and `PccDashboardCard` unless the wrapper is itself an intentionally-tested grid child (plan §3.3).
- Fake affordances — non-working search/action/control must not appear interactive without a clear preview/disabled/deferred state and reason (plan §3.6 / §11).
- Sage mutation/sync/posting/update/writeback implication — Cost & Time Sage cue must remain book-of-record / no-writeback (plan §3.5 / §8.7).
- Dependency creep — no new runtime dependencies; no `echarts-for-react`; no icon libraries (plan §3.4).
- Broad / global CSS resets — only scoped PCC primitives, tokens, component contracts, or targeted layout fixes (plan §3.7).
- Raw one-off color introductions — use existing PCC tokens, theme variables, or intentionally-documented token additions (plan §3.7, §5.6).
- Redundant top-level bento card reintroduction — prohibited where the card only repeats hero/header (plan §3.8 / §7.3).
- Source-status drift — never replace `available` with `live`; never overload existing fields like `riskTag` to carry origin metadata (memory: feedback_no_overload_existing_field_for_origin_metadata).
- Test weakening — do not weaken tests to pass; do not test CSS module class names as behavior contracts; preserve `[data-*]` markers (plan §3.7).
- UI strings containing `mock`, `placeholder`, `TODO`, `fixture`, `demo`, prompt numbers, wave names, or implementation sequencing (plan §3.6).
