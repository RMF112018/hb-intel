# Wave 14 Wireframes — Approvals / Checkpoints

## Purpose

These wireframes define the major screen groups required to implement Phase 14. They are text-based developer wireframes intended for documentation and local-code-agent execution, not pixel-perfect design comps.

## Screen Groups

1. Approvals Home
2. My Approvals
3. Approval Detail
4. Checkpoint Registry
5. Decision History
6. Escalation Queue
7. Admin Verification Queue
8. Module Integration Panels

## Shared Wireframe Symbols

- `[KPI]` — summary metric card.
- `[Filter]` — filter chip, dropdown, or facet.
- `[Row]` — queue row.
- `[Panel]` — right/side panel or drawer.
- `[Action]` — button/action affordance.
- `[Disabled: reason]` — visible disabled action with explanation.
- `[Evidence]` — evidence/source card.
- `[Audit]` — history/event row.
- `[HBI]` — HBI summary area with no-authority warning.

## UX Priorities

- Action clarity.
- Evidence visibility.
- Authority clarity.
- Downstream impact visibility.
- Redaction awareness.
- Keyboard accessibility.
- No direct external-system execution.
