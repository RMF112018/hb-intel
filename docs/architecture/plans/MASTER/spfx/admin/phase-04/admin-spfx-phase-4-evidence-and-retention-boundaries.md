## 1. Authoritative evidence strategy

Evidence is metadata-first. `IAdminEvidenceReference` records track what was produced, when, by which run/step, and where the payload lives. Payloads are stored inline when small or referenced by `storageLocator` when large.

| Store | Table | Purpose | Phase |
|-------|-------|---------|-------|
| Evidence metadata | `AdminEvidence` (Table Storage) | Track evidence references with retention class | P4-06 (implemented) |
| Inline payloads | `AdminEvidence` entity `inlinePayloadJson` field | Small payloads (< 32 KB) stored directly | P4-06 (implemented) |
| Offloaded payloads | `admin-evidence` blob container | Large payloads stored as blobs | Phase 6 (deferred) |

---

## 2. Inline vs offloaded thresholds

| Rule | Value | Rationale |
|------|-------|-----------|
| Inline threshold | 32,768 bytes (32 KB) | Azure Table entities have 1 MB limit; 32 KB leaves ample room for metadata fields |
| Decision function | `isEvidenceInlineable(payload)` | Serializes payload to JSON, checks byte length |
| Offload reference | `blob://admin-evidence/{runId}/{evidenceId}.json` | Generated by `generateBlobLocator()` |
| Null/undefined payload | Always inlineable (no payload to store) | Metadata-only evidence references are valid |

## 3. Retention metadata model

| Field | Type | Values | Stored on |
|-------|------|--------|-----------|
| `retentionClass` | `EvidenceRetentionClass` | `'operational'` \| `'compliance'` \| `'permanent'` | `AdminEvidence` entity |

| Class | Default TTL | Examples | Enforcement |
|-------|-------------|---------|-------------|
| `operational` | 90 days | Step progress, preview results, probe snapshots | Deferred to Phase 13 |
| `compliance` | 365 days | Run input/output, operator decisions, config traces | Deferred to Phase 13 |
| `permanent` | Indefinite | Destructive-action evidence, post-run validation | Deferred to Phase 13 |

Phase 4 stores the classification on every evidence record. Phase 13 enforces TTL-based cleanup.

## 4. Export / retrieval boundary

| Concern | Rule |
|---------|------|
| Evidence listing | `GET /api/admin/runs/{runId}/evidence` returns evidence refs (P4-05) |
| Evidence detail | `getEvidence(evidenceId)` returns the reference; inline payload available on entity |
| Offloaded payload retrieval | Not implemented (Phase 6) ‚Äî `storageLocator` indicates blob reference |
| Payload redaction | No automatic redaction in Phase 4; operator sees what was captured |
| Export format | Not implemented ‚Äî future phases define export packaging |
| Sensitive data | Callers should avoid storing secrets in evidence payloads; adapter-specific data should be sanitized before evidence capture |

---

## 5. Deferred policy-enforcement items

| Item | Target phase |
|------|-------------|

| TTL-based retention cleanup | Phase 13 |
| Evidence redaction (right-to-forget) | Phase 13 |
| Evidence export packaging | Phase 6 |
