# Plan Summary — Wave 01

## Goal
Create the minimum architectural truth required for a world-class rebuild.

## Sequence
1. Reclassify Safety in the homepage shell
2. Redefine the safety information architecture and content contract
3. Lock application-level mode and shell-fit rules

## Exit criteria
- Safety is no longer governed as a newsroom-adjacent supporting passenger by default
- the data contract can express a meaningful “what matters now” safety story
- the application exposes explicit `standard` / `compact` / `minimal` behavior expectations


## Checklist emphasis
This wave is primarily accountable for improving:
- Doctrine and host compliance
- Purpose-fit sophistication and persona expression
- Homepage integration quality
- Breakpoint and shell-fit quality
- Contract, data, and backend seam rigor

## Wave gate
Do not mark Wave 01 complete unless the closure note explicitly states how the changes improve the scorecard and which sub-2 categories still remain open for Wave 02/03.
