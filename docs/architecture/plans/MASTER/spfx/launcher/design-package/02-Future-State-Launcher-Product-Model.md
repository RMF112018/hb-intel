# 02 — Future-State Launcher Product Model

## Product definition

The homepage launcher is a **compact premium execution rail** with one secondary disclosure path.

It has only three user-facing states:

1. **Default row state**
2. **Drawer open state**
3. **Handheld collapsed trigger state**

## Default row state

The row is:
- compact
- premium
- legible
- immediately scannable
- icon-forward
- confidence-building

Users should immediately understand:
- these are their primary launch destinations
- the rightmost orange tile is the overflow entry
- the row is part of the homepage’s first-screen execution model

## Drawer open state

The drawer is:
- a bottom-sheet utility surface
- horizontally scrollable
- spatially simple
- visually clean
- free of secondary IA clutter

The drawer is not:
- a category browser
- a modal settings panel
- a stacked grid of subsections
- a place for explanatory copy

## Handheld collapsed trigger state

On handheld breakpoints, the row becomes a single full-width trigger that opens the same bottom drawer.
That trigger must remain aligned with the homepage width rhythm and inherit the same product language.
