# Prompt Change Log

## Baseline prompt disposition

### Prompt 01 — Rebuild the editorial workspace shell
**Decision:** Keep, but rewrite substantially.

**Why:**
The shell is still under-premium, but the repo already contains a real sectioned canvas, queue rail, section index, preview section, and readiness rail. The old prompt overstates how much of the shell is still absent and understates what should now be preserved.

### Prompt 02 — Replace hardcoded styling with a governed Publisher surface system
**Decision:** Keep, but rewrite substantially.

**Why:**
The repo already contains a Publisher token seam under `sharedChrome/tokens.module.css`, but adoption is incomplete and feature-local CSS still carries hardcoded values and stale pre-primitive class residue. The new prompt focuses on completion and cleanup, not speculative system invention.

### Prompt 03 — Upgrade iconography, avatars, and micro-interactions
**Decision:** Keep, expand, and tighten.

**Why:**
This prompt remains strongly repo-truth accurate. Pseudo-icon patterns, initials avatars, and low-ambition toolbar affordances still exist. The new prompt adds sharper dependency guidance, accessibility expectations, and exact surfaces to scrub.

### Prompt 04 — Recompose preview and readiness into a stronger trust loop
**Decision:** Keep, but rewrite substantially.

**Why:**
The repo already has separate preview and readiness seams plus diagnostics. The problem is no longer “they do not exist,” but that they are not yet composed into the fastest, clearest author trust loop.

### Prompt 05 — Rebuild team and gallery surfaces for editorial use
**Decision:** Keep, but rewrite substantially.

**Why:**
The repo already has visual management surfaces, people-picker-backed team composition, composer flyouts, and invariant enforcement. The remaining work is product-grade completion, label cleanup, better foreshadowing of published output, and better interaction polish.

## New prompt added

### Prompt 06 — Close author-facing label governance and selector accessibility
**Decision:** Add.

**Why:**
The baseline package omitted a now-material dependency for true Wave 01 closure. The repo already has centralized label governance, but some raw values still leak, and key selector interactions need stronger accessibility and author-trust treatment. This work materially affects the same Wave 01 product seams and should not be deferred.

## Prompt merge / split decisions

- No baseline prompt was removed outright.
- No baseline prompt was merged away.
- A new sixth prompt was added because the missing work was too important to bury inside another prompt, but still tightly bounded enough to avoid turning Wave 01 into a broad future-wave package.
