# Publisher UI remediation — Wave 02

## Wave posture
This package is written to match the **Wave 01 structural rebuild framing**.

That means every prompt in this wave assumes the default response to a weak authoring seam is **structural replacement or structural refactoring**, not decorative polish, copy-only adjustment, or light interaction cleanup.

## Wave objective
Close the remaining cross-layer workflow burdens that still make the Publisher feel like the author is working around the tool.

Wave 02 is responsible for the rebuild and closure of:
- governed media acquisition for single-image and gallery flows
- truthful local draft resilience, recovery, and navigation-loss protection
- editorial-grade body composition ergonomics
- inline guidance and preview trust bridge closure
- final accessibility, host-fit, and hardening gaps in the touched seams

## Structural rebuild rules for this wave
- Do **not** treat weak seams as acceptable merely because a partial implementation exists.
- Do **not** preserve URL-first, save-at-risk, or guidance-at-a-distance interaction models because they already compile.
- Do **not** solve author friction with copy changes alone when the interaction model itself is weak.
- Do **not** reopen unrelated Wave 01 shell/layout decisions unless a direct dependency makes it necessary for Wave 02 closure.
- Do **not** defer meaningful closure work that belongs inside this wave.

## Execution order
Work the prompts in numerical order.
