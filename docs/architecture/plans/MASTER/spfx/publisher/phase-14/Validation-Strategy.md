# Wave 01 Validation Strategy — Structural Redesign Edition

Every prompt must be closed with repo-truth evidence, not subjective “looks better” claims.

## Universal validation requirements

For each prompt:

1. record the exact seams changed
2. state what was preserved
3. state what was structurally replaced or recomposed
4. validate the affected states in the running Publisher
5. run targeted tests where they exist
6. add or update tests when governed helpers, accessibility semantics, invariants, or interaction-critical behavior change
7. create the required closure note before moving on

## Structural-redesign rule

A prompt is **not closed** if the result can honestly be described as:

- refinement
- cleanup
- polish
- simplification only
- modest re-layout
- disclosure wrapping over the same weak composition

## Prompt-specific proof expectations

### Prompt 01
Prove the shell no longer reads as three adjacent admin cards.

### Prompt 02
Prove first-pass drafting uses a materially shorter visible decision path.

### Prompt 03
Prove project binding still works and no raw internal project identity is first-order.

### Prompt 04
Prove hero/gallery/media authoring is no longer primarily URL-centric data entry in the normal path.

### Prompt 05

### Prompt 06
Prove legacy/operator content is still truthful but clearly quarantined from the normal author path.

### Prompt 07
Prove surface language, spacing rhythm, and tokenized visual policy read as one deliberate product system.

### Prompt 08
Prove label governance, selector semantics, keyboard support, and icon/microinteraction quality are materially stronger.
