# Wave 01 Plan Summary — Structural Redesign Closure

## Objective

Rebuild the HB Publisher UI into a **world-class SharePoint editorial authoring product** by forcing structural redesign in the seams where the live repo still behaves like a technically improved admin workbench.

Wave 01 is not complete when the UI is merely cleaner.
Wave 01 is complete only when the rendered experience is materially re-authored around:

- faster first-pass drafting
- stronger editorial hierarchy
- lower cognitive load
- clearer publish confidence
- premium, governed interaction language
- truthful but non-dominant system narration

## Repo-truth posture

The live repo already contains meaningful assets that must be preserved where they are good:

- a three-region shell
- a sectioned authoring canvas
- a real project lookup seam
- a TipTap body editor
- separate preview and readiness controllers
- team and gallery management surfaces
- a Publisher shared-chrome seam
- author-facing label helpers

The problem is **not absence**.
The problem is that the current composition still leaves too much of the product trapped in a utility-grade information architecture.

## Final closure sequence

1. structurally redesign the shell and page information architecture
2. structurally redesign the default authoring flow and field hierarchy
3. structurally redesign project selection and bound-project presentation
4. structurally redesign media acquisition plus hero/gallery authoring
5. structurally redesign preview/readiness/publish mapping
6. structurally redesign legacy/operator separation and advanced disclosure posture
7. structurally redesign the visual system and eliminate timid surface language
8. structurally redesign labels, selector accessibility, and icon/microinteraction quality

## Success standard

Wave 01 succeeds only if the Publisher becomes materially stronger in all of the following ways:

- the center authoring experience is clearly dominant
- the default author path feels short and intentional
- critical decisions are front-loaded and advanced decisions are clearly secondary
- raw/internal system language is no longer first-order author experience
- media authoring no longer behaves like URL-centric data entry
- readiness and preview form one fast trust loop
- the product reads as one authored editorial workspace rather than adjacent admin cards
- no meaningful regression is introduced to data truth, save/publish gating, preview composition, or invariants

## Explicit no-deferral rule

If a defect materially harms product identity, authoring speed, trust, accessibility, media quality, publish confidence, or visual-system integrity inside these seams, it belongs in Wave 01 and must be addressed now.
