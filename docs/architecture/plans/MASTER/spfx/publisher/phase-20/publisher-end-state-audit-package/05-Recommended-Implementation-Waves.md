# Recommended Implementation Waves

## Wave 01 — Live-runtime closure and authoritative seam hardening

### Purpose
Close the gaps that most directly undermine the Publisher’s claim to be low-friction, trustworthy, and product-grade in real production use.

### Scope
1. wire governed asset-library acquisition through the live SPFx mount/runtime boundary
2. harden the project lookup contract beneath the current picker UX
3. expand meaningful burden-reducing defaults and first-pass assistance
4. tighten first-pass product behavior so less ordinary workflow depends on helper narration

### Preserve explicitly
- current three-region shell
- current readiness and save-trust model
- current project picker interaction model
- current team and gallery product surfaces
- current system-managed slug posture

### Closure standard
Wave 01 is closed only when the live production posture is materially more authoritative and lower-friction, not merely better documented.

---

## Wave 02 — Flagship editorial refinement

### Purpose
Advance the Publisher from a strong product foundation to a more obviously world-class editorial experience without disturbing the structural gains already achieved.

### Scope
1. assess and implement only justified story-editor improvements
2. deepen preview fidelity where it materially improves author confidence
3. refine advanced editorial workflows only where they map cleanly to rendered output and real author needs
4. continue removing narrated friction in favor of product behavior

### Preserve explicitly
- governed schema posture unless real output needs justify expansion
- preview/readiness trust coherence
- low-friction first-pass workflow
- current Project Spotlight present-state scope

### Closure standard
Wave 02 is closed only when the Publisher feels more premium and more inevitable to use, not merely more feature-rich.

---

## Why two waves are justified

Two waves are warranted because:
- Wave 01 deals with live-runtime truth and authoritative seam quality
- Wave 02 should build on those stronger foundations rather than competing with them
- this sequencing reduces rework and keeps the shell stable while high-value gaps are closed first
