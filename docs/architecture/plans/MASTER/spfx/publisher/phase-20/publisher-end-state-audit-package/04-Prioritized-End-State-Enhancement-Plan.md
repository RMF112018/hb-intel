# Prioritized End-State Enhancement Plan

## Priority 1 — Make governed asset acquisition real in production
**Gap closed:** designed-but-not-fully-wired media governance  
**Impact:** High  
**Timing:** Implement now  
**Type:** Targeted refinement with cross-layer implications

### Product-grade outcome
- live production media authoring leads with tenant-safe asset browse
- raw URL entry becomes truly secondary
- hero, secondary, and gallery acquisition all share the same governed source posture
- alt-text seeding and asset metadata support become more trustworthy

### Cross-layer implications
- mount/runtime seam
- asset search adapter
- possible SharePoint or curated-library lookup
- image/media composer surfaces
- proof-of-closure testing in live host-fit context

## Priority 5 — Advance story authoring only where justified by real output needs
**Gap closed:** editorial surface not obviously final-state flagship answer  
**Impact:** Medium  
**Timing:** Later wave after runtime/data seam hardening  
**Type:** Refinement, potentially structural

### Product-grade outcome
- story authoring evolves only in ways that clearly improve published output and author confidence
- no unnecessary editor bloat
- stronger parity between authoring and rendered page experience

### Cross-layer implications
- story editor schema
- preview fidelity
- publishing templates
- author workflow expectations

## Recommended prioritization logic

1. close live-runtime integration gaps first
2. harden authoritative data seams second
3. then reduce author burden further
4. then deepen flagship editorial capability on top of the stabilized foundation
