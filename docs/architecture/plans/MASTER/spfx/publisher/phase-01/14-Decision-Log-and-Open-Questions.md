# 14 — Decision Log and Open Questions

## Purpose

Use this file as the persistent operating log while executing the prompt series.

## Decisions to capture

Record the final answer for each of these as implementation proceeds:

1. Which repo package/app owns the publisher UI?
2. Which service layer owns publish orchestration?
3. Where do the control-plane lists live?
4. How is the XML shell referenced at runtime or publish time?
5. What is the page slug strategy?
6. What exactly triggers regeneration rather than in-place republish?
7. What is the archive/withdraw policy?
8. Which Team Viewer properties are treated as stable MVP contract?
9. Which template/profile fields are still intentionally extensible?
10. Which items are explicitly deferred to Wave 2?

## Open-question discipline

Do not leave major implementation questions floating informally in chat notes.

Capture them here with:
- question
- current assumption
- blocking or non-blocking status
- final decision
- impact area

## Closure note template

At the end of each prompt, add a short note with:
- what changed
- what was validated
- what remains open
- whether the next prompt can begin safely
