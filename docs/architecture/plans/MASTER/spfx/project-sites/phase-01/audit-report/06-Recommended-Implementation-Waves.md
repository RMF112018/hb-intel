# 06 — Recommended Implementation Waves

## Wave 01 — Authority and data-trust foundation
Purpose:
- remove dead host/config semantics
- restore authoritative year resolution
- strengthen the repository adapter
- introduce an explicit launch-state model

Why first:
These changes establish the trust model the rest of the UX depends on.

Contained work:
1. End-to-end year-context contract and runtime-config plumbing
2. Canonical Project Sites repository adapter with explicit field contract
3. Explicit launch-state derivation and card-state rewrite
4. Truthful partial-data / no-context messaging

## Wave 02 — Responsive maturity and confidence polish
Purpose:
- harden compact-mode behavior
- strengthen scanning confidence
- improve people/access cues
- align docs and validation with the new future-state standard

Why second:
These changes should build on the stabilized authority and state model from Wave 01.

Contained work:
1. Breakpoint contract and compact-mode control-bar redesign
2. Card identity enrichment and access-confidence cues
3. People display-label improvement
4. Validation / docs / closure evidence updates for the new standard
