# Project Sites End-State Wave 01

## Goal
Close the foundational trust gaps that currently prevent Project Sites from functioning as an authoritative launch surface.

## Sequence
1. Restore year/context authority and remove dead configuration semantics
2. Stabilize the repository adapter and explicit field contract
3. Replace coarse launch-state inference with an explicit model
4. Add truthful partial-data and no-context messaging

## Rules for this wave
- Treat the live repo as source of truth
- Do not redesign unrelated surfaces
- Do not re-read files that are still in active context unless needed to confirm drift, dependencies, or uncertainty after changes
- Do not leave any meaningful in-scope trust gap partially implemented
