# Prompt 08 — Composition, Integration, and Final Polish

## Objective

Perform the final whole-page integration pass so HB Central reads as one product instead of a collection of individually improved cards. This phase must resolve composition rhythm, shell/homepage handoff, zone hierarchy, and final polish.

## Scope

Primary targets:

- homepage composition references
- zone wrappers / shared layout files
- any final `@hbc/ui-kit` touchpoints
- any shell/homepage integration edges
- docs and doctrine updates required by the completed redesign

## Current-State Problem to Solve

Even if individual webparts improve, the homepage can still fail if the full-page composition remains too even, too stacked, or too SharePoint-hosted in feel. This final phase must fix that.

## Hard Gates

- Do **not** reread files already in your active context or memory.
- Do **not** stop at per-webpart improvement if the whole page still feels disconnected.
- Do **not** accept equal-weight composition across the full page.
- Do **not** leave the shell/homepage handoff visually weak.
- Do **not** claim completion without rendered proof.

## Required Outcomes

The final composed homepage should:

- have a convincing top-to-bottom narrative
- clearly prioritize the top band
- make utility/discovery feel purposeful
- make communications and operations feel intentionally placed
- feel authored inside SharePoint rather than merely hosted by it

## Implementation Requirements

1. Audit the fully rendered page after prior phases.
2. Adjust zone spacing, contrast, and hierarchy where the page still feels too even.
3. Tune shell/homepage transition points.
4. Remove remaining weak or repetitive patterns.
5. Update doctrine / docs to reflect the final ownership and visual-system reality.
6. Capture proof of the final rendered state.

## Validation

Show proof that:

- the page now reads as one cohesive product
- the top band, utility, editorial, operational, and discovery zones each have appropriate identity
- the shell no longer weakens the page
- there are no obvious rhythm or hierarchy failures left
- packaging, import discipline, accessibility, and reduced-motion behavior remain intact

## Output Format

Return:

1. final summary of the homepage reset
2. files changed
3. documentation updates made
4. final rendered-state proof
5. remaining gaps, if any

## Final Instruction

Do not soften this work.

The current homepage is not acceptable. The goal of this phase is to produce measurable visual improvement that can be seen immediately in the rendered experience, not merely explained in code review.
