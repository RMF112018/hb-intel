# Prompt 09 — Full-Page Composition, QA, and Visual Closure

## Objective

Perform the final composition pass that turns the redesigned shell and webparts into one coherent premium homepage product.

## Primary Scope

- full homepage composition across Lane A and Lane B
- any remaining shared-kit adjustments needed to resolve rhythm, spacing, or hierarchy conflicts
- docs and closure artifacts

## Hard Instructions

- Do not reread files already in current context or memory.
- This phase is not optional polish. It is where the page either becomes one product or stays a collection of modules.
- Do not conclude the work while obvious white-space imbalance, hierarchy breaks, or same-ness remain.

## Required Work

1. Review the redesigned full page in realistic SharePoint-hosted layout.
2. Resolve:
   - cross-zone rhythm
   - section-to-section transitions
   - shell-to-homepage handoff
   - visual weight balance
   - hierarchy conflicts
   - edge alignment
   - responsive behavior
   - interactive consistency
3. Remove any remaining evidence that the homepage is just a set of similar white cards.
4. Confirm the page now has a convincing focal sequence from:
   - shell
   - top band
   - command/utility
   - discovery
   - communications
   - operations
5. Update any README, doctrine, or phase docs required by the new end state.
6. Produce final visual evidence and closure documentation.

## Required Validation

- final before/after full-page screenshots
- accessibility verification
- reduced-motion verification
- focus-state verification
- SharePoint-hosted realism verification
- statement of what changed in perceived product quality

## Deliverables

- final integration changes
- final doc updates
- final closure note
- visual evidence package

## Completion Bar

This phase is complete only when the homepage no longer reads as “custom cards inside SharePoint” and instead reads as a deliberate, premium, branded internal product.
