## Named Failing Seams

### Seam 1 — mount.tsx WEBPART_RENDERERS (CRITICAL)

**Location:** `apps/hb-webparts/src/mount.tsx` lines 24-35

The `WEBPART_RENDERERS` map registered 10 webpart IDs but **omitted** `28acd6a7-2582-4d8a-86d4-b52bfbeb375c` (HbSignatureHero). When the SPFx shell loaded HbSignatureHero via its manifest, the shell entry shim correctly called `mount()` with `webPartId: "28acd6a7-..."`, but the mount function could not find a renderer. The fallback on line 60 rendered `ReferenceHomepageComposition` (the full 5-zone homepage preview) instead of the standalone hero.

**Impact:** SharePoint rendered the wrong component — a reference composition instead of the rebuilt hero.

**Fix:** Add HbSignatureHero import and renderer entry to `WEBPART_RENDERERS`.

### Seam 2 — Packaged version number (COSMETIC)

**Location:** `tools/spfx-shell/config/package-solution.json`

The build orchestrator (`tools/build-spfx-package.ts`) resets `package-solution.json` during domain builds. The emitted `.sppkg` AppManifest.xml and feature XML showed `Version="1.0.0.65"` (the pre-Phase-01 value) instead of the current version. This is cosmetic — it doesn't affect runtime behavior — but it means the package version doesn't reflect Phase 01 work.
