# Prompt 03 — Surface Reset Memo and Locked Target Contract

## Objective

Turn the repo-truth audit and gap map into a locked target contract for the Tool Launcher / Work Hub so downstream phases are implementing one clear surface model.

## Working rules

- repo truth first
- do not re-read files still in your current context unless needed
- do not code beyond minimal documentation updates unless essential
- do not reopen already-solved schema ideation
- keep the target aligned with the homepage doctrine and the architecture brief
- keep the launcher compatible with the Signature Hero quality bar without making it a second hero

## What you must produce

Create a markdown file named:

- `phase-00-surface-reset-and-target-contract.md`

It must define the locked target in these sections:

### 1. Locked Role of the Webpart
State exactly what the launcher is in homepage terms.

### 2. Locked Composition Model
Define the command band, flagship stage, utility rail, workflow shelves, and all-platforms layer at the appropriate level of specificity.

### 3. Locked Content-Source Principle
State that SharePoint list content is the source of truth and the architecture brief remains the UI hierarchy source.

### 4. Locked Data-Seam Principle
State that the current grouped-config seam is transitional and must not remain the long-term contract.

### 5. Locked Shared-vs-Local Boundary
Define what should remain local to homepage work and what may be promoted later if justified.

### 6. Locked Exclusions
State what the next phases should not do.

### 7. Next-Phase Handoff
Define the exact expected objective of Phase 01.

## Additional instruction

Also update any local planning/readme note in the working branch if there is a clearly appropriate project-local place to record the result of this phase. If there is no appropriate place, say so and do not invent one.
