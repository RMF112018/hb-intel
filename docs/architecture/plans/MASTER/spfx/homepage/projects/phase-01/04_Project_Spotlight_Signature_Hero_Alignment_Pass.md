# Prompt 04 — Project Spotlight Signature Hero Alignment Pass

## Objective

Refine Project Spotlight so it clearly belongs to the same premium homepage family as the Signature Hero without becoming visually repetitive.

This phase is a quality-bar pass, not a broad architecture rewrite.

## Prerequisite

Complete Prompts 01 through 03 first.

Do **not** re-read files already in your current context or memory unless they changed, your context is stale, or the task scope expanded.

## Required Comparison Scope

Use the current Signature Hero implementation path as the comparison benchmark.

Audit and compare:

- width usage
- spacing rhythm
- typography authority
- image treatment
- surface treatment
- motion restraint
- overall polish level

## Required Outcome

After this prompt, Project Spotlight should feel:

- premium
- image-led
- brand-forward
- calm
- disciplined
- compatible with the Signature Hero

It should **not** feel:

- visually unrelated
- overly decorative
- too card-like
- too busy
- too flat

## Specific Implementation Work

### 1. Tune spacing rhythm
Make Spotlight’s internal spacing cadence feel intentional and premium.

### 2. Tune typography
Adjust title, headline, summary, metadata, and CTA hierarchy so the feature reads with authority.

### 3. Refine image treatment
Ensure images feel rich and deliberate, not generic or boxed-in.

### 4. Reduce unnecessary chrome
Remove anything that weakens the editorial feel:
- noisy borders
- gratuitous containers
- badge overload
- dashboard-like separators

### 5. Align motion posture
Ensure hover and subtle transitions remain restrained and consistent with the homepage premium system.

## Required Deliverables

### A. Hero Alignment Summary
Explain exactly how Spotlight now aligns with the Signature Hero.

### B. Quality Bar Delta
Describe what changed from the pre-alignment version and why it was necessary.

### C. Residual Gaps
State any remaining premium-quality gaps that should be addressed in later phases.

## Validation Requirements

Before closing:

- perform a direct comparison against the Signature Hero path
- verify Spotlight does not visually compete with or duplicate the hero
- verify the surface still reads clearly with realistic project data
- run the smallest credible lint/type/test validation for the touched scope

## Risk Exposure

Watch for:
- over-polishing into gimmick territory
- decorative excess
- too much visual similarity to the hero
- under-polished typography
- hero-level density copied where it should only be echoed

## Standards / Best Practices

- compatible, not identical
- premium, not flashy
- stronger typography, less chrome
- restrained hover and motion
- editorial, not dashboard

## Final Instruction

Do not introduce deeper data-model logic in this prompt.

This prompt is complete only when Project Spotlight credibly belongs in the same homepage premium system as the Signature Hero.
