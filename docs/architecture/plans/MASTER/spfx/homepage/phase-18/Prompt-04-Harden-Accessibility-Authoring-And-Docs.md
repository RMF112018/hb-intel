# Prompt 04 — Harden Accessibility, Authoring Safety, Validation, and Doctrine for the New Signature Hero

## Objective

Finish Phase 01 by hardening the rebuilt Signature Hero and updating doctrine/documentation so the new canonical minimal hero direction is explicit and durable.

## Important Working Rule

**Do not re-read files that are already in your active context window or current working memory.**
Only re-open files when necessary to validate or edit with precision.

## Required Work

### 1. Accessibility hardening
Validate and correct as needed:
- text/background contrast
- keyboard/focus behavior for any remaining interactive affordances
- reduced-motion handling
- responsive reflow
- zoom resilience
- screen reader clarity for hero content

### 2. Authoring safety
Ensure the hero behaves professionally when:
- minimally configured
- partially configured
- background media is missing
- personalized identity fields are incomplete or fallback-driven
- viewed in edit mode

If authoring configuration is reduced because the hero now has minimal locked content, make the new expectations clear in code comments and docs.

### 3. Reference composition alignment
Ensure the homepage reference composition reflects the new reduced canonical hero and no longer implies obsolete hero furniture.

### 4. Doctrine / docs update
Update the relevant doctrine/reference docs to reflect the new canonical Phase 01 hero direction.

At minimum, update guidance so it is clear that the flagship homepage hero:
- is the unified signature hero
- contains only logo, “Build with GRIT.”, and personalized welcome
- intentionally rejects the older split greeting/editorial model
- intentionally rejects the old gradient-heavy treatment
- uses a premium restrained background approach instead

### 5. Completion notes
Produce concise completion notes summarizing:
- what changed
- how the hero is now governed
- what remains for later phases
- any open risks

## Hard Rules

- Do not reopen or widen scope into unrelated homepage zones.
- Do not add back removed hero content.
- Do not leave docs contradicting implementation.
- Do not call the phase complete without explicit validation.

## Validation Checklist

You are done only when you can show:
- canonical hero path is unified
- hero content is reduced to the three locked elements
- gradient-heavy treatment is gone
- full-width intent remains intact
- hero still feels premium, substantial, and deliberate
- docs match repo truth
