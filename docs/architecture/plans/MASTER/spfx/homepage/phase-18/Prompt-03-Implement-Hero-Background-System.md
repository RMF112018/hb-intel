# Prompt 03 — Replace the Gradient With a Premium Background System for the Signature Hero

## Objective

Remove the current gradient-heavy hero background treatment and replace it with a more appropriate premium background system.

The user strongly dislikes the current gradient because it feels outdated and stodgy.

Your job is to replace it with a background direction that feels:
- current
- premium
- architectural
- restrained
- credible for a sophisticated construction brand
- materially more elegant than the current orange/blue wash

## Important Working Rule

**Do not re-read files that are already in your active context window or current working memory.**
Only revisit files if needed to implement or validate this prompt accurately.

## Locked Background Direction

Implement the background around this art-direction brief:

### Primary background direction
A **quiet, cinematic, real-material or real-project backdrop** that behaves more like premium wallpaper than a graphic effect.

Target characteristics:
- dark neutral / charcoal / graphite base tonality
- restrained steel-blue influence is acceptable
- subtle warmth may exist in the photography/material itself, but **not** as a bold color gradient wash
- minimal visual noise
- no “sunset blend”
- no obvious enterprise gradient banner treatment
- no neon, glow, or startup-tech styling

### Preferred implementation hierarchy
Use the best repo-appropriate approach available, in this order:

1. **Real photography art direction**  
   Use a high-quality, wide, low-clutter project or craft image if an appropriate approved asset already exists or can be cleanly wired into the current hero authoring model.
   Preferred subject matter:
   - architectural detail
   - material texture
   - elegant project photography
   - construction craft detail
   - structural rhythm
   - premium built-environment imagery

   Avoid:
   - busy jobsite chaos
   - group photos
   - stock-office clichés
   - obvious marketing fluff
   - images with text baked into them
   - images that compete with the greeting

2. **Textured premium fallback surface**  
   If a real image is not available or not appropriate as the default implementation, build a premium fallback surface using:
   - deep near-solid base color
   - subtle material/noise texture
   - restrained shadowing / vignetting / depth cues
   - extremely subtle structural linework or atmospheric layering only if it improves the composition

   This fallback must **not** read as a generic flat fill or cheap synthetic effect.

## Background System Requirements

### 1. Readability
- text must remain readable at realistic responsive crops
- use a non-flashy readability strategy
- avoid obvious heavy overlay bands if a subtler solution will work

### 2. Responsiveness
- test crop behavior across wide and narrower homepage states
- keep focal zones safe for the logo, tagline, and greeting
- do not allow critical background subjects to collide with text

### 3. Minimalism support
The new background must help the reduced-content hero still feel substantial.
It should provide depth and tone without demanding attention.

### 4. No gradient regression
Do not implement another large color-wash gradient under a new name.
Subtle tonal layering that supports a photographic or material background is acceptable.
The final result must not read as “the same gradient problem, slightly changed.”

## Suggested Implementation Tactics

Use whatever is most appropriate in repo truth, but likely candidates include:
- authored image support with better default treatment
- CSS module-based background layers
- subtle texture/noise asset or inline SVG texture
- controlled pseudo-elements for depth
- more disciplined overlay logic

## Hard Rules

- no stodgy blue/orange gradient hero wash
- no loud abstract blobs
- no glossy tech-demo background
- no cluttered or literal construction collage
- no accessibility regression
- no performance-hostile background implementation

## Deliverables

Produce:
1. the new hero background system
2. concise completion notes covering:
   - chosen background direction
   - why it is more premium/current
   - fallback strategy
   - any asset dependency or authoring implication
