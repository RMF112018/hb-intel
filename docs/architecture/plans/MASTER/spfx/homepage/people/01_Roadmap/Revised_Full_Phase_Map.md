# Revised Full Phase Map — People & Culture with Kudos

## Objective

Define the complete implementation sequence for the fully merged People & Culture + Kudos feature.

## Phase sequence

### Phase 00 — Doctrine Lock and Surface Reset
Lock the final target architecture and reject invalid implementation paths.

### Phase 1A — Content Model and Source Strategy
Define the unified source model for celebrations, announcements, Kudos submissions, approval state, and recipient types.

### Phase 1B — Visibility, Persistence, Approval, and Editorial Rules
Define the business rules that determine what is visible, when, why, and in what order.

### Phase 2 — Adapter and Output Foundation
Define the normalized output structure for:
- Band A
- Kudos module
- Band B

### Phase 3 — Desktop Composition Skeleton
Build the desktop shell:
- header
- Band A shell
- Kudos shell
- Band B shell

### Phase 4 — Band A — Highlights / Announcements
Build the formal editorial milestone layer.

### Phase 5 — Kudos Homepage Module
Build the embedded homepage recognition engine.

### Phase 6 — Dedicated Kudos Page + HR/Admin Moderation Workspace
Build the archive, long-form submission, and moderation experience.

### Phase 7 — Band B — This Week
Build the compact weekly celebration layer beneath Kudos.

### Phase 8A — Responsive Behavior
Adapt the complete three-part composition to tablet/mobile.

### Phase 8B — Authoring / Loading / Empty / Partial-Data Hardening
Make the complete surface safe in real SharePoint authoring and imperfect data conditions.

### Phase 9 — CTA, Editorial Controls, and Final Refinement
Add the final polish layer.

### Phase 10 — Packaging, Runtime Proof, and Production Hardening
Validate the completed feature through the existing hb-webparts package lane and SharePoint-hosted runtime.
