# Prompt-03 — HB Kudos Public Composer Flow and Form Experience

```md
Objective

Conduct a focused UI-only implementation pass on the **public HB Kudos composer experience** so the submission flow feels more guided, compact, premium, and recognition-oriented rather than operational and form-heavy.

Primary Intent

The composer should feel like a warm recognition flow, not like an admin workflow form placed in a premium shell.

Critical Constraints

- Use live repo truth.
- Do not begin by re-reading files already in active context or memory.
- Maintain strict compliance with `@hbc/ui-kit` and homepage/SPFx doctrine.
- Promote reusable or durable UI into the correct shared layer where appropriate.
- Keep this prompt centered on composer UX and presentation, not on backend workflow changes.

Focus

Improve the composer in the areas of:
- overall shell rhythm and structure
- opening experience and introductory content density
- sequence and grouping of fields
- helper-copy load
- visual weight of recipient buckets vs message-entry fields
- action hierarchy in the footer
- sense of progress and task momentum

Target Outcomes

The rendered composer should move toward these outcomes:
- less redundant intro framing
- less box-stack fatigue
- faster path to meaningful action
- stronger warmth and trust
- better hierarchy between recipient entry, recognition content, and final action
- improved perceived polish and efficiency

Implementation Freedom

You may take different implementation approaches, including but not limited to:
- changing the composer information architecture
- reducing or reworking the intro treatment
- changing how sections are grouped or disclosed
- tightening spacing and hierarchy
- introducing a more guided or staged rhythm if appropriate
- improving preview or progressive disclosure if that is the best fit

Do not over-prescribe a specific flow if repo truth suggests a stronger alternative.

Do Not

- keep the existing long stacked form experience with only cosmetic restyling
- preserve redundant content that adds height without adding value
- leave the composer as a premium-colored workflow form

Deliverables

1. Implement the composer UX upgrade.
2. Update or promote shared UI pieces where warranted.
3. Provide a concise summary of what changed.
4. State which remediation-matrix rows were advanced or closed.
5. Call out what remains for Prompt-04 and Prompt-05.

Acceptance Standard

This prompt is successful only if the composer experience feels materially more guided, compact, and premium, with a clearer sense of flow and less visual/form fatigue.
```
