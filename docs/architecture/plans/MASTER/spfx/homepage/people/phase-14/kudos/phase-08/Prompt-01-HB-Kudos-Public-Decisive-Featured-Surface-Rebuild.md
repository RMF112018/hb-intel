# Prompt-01 — HB Kudos Public Decisive Featured Surface Rebuild

```md
Objective

Restore the public HB Kudos featured recognition surface through a **decisive rebuild or replacement** if necessary.

Primary problem

The current featured recognition area is still rendering as a hollow or broken shell.

Prior prompts that attempted softer contract/rendering alignment did not produce sufficient user-visible change.

This prompt is therefore explicitly more forceful.

Critical Constraints

- Use live repo truth.
- Do not begin by re-reading files already in active context or memory.
- Maintain strict focus on the **public featured recognition surface**.
- Maintain strict compliance with `@hbc/ui-kit`, homepage doctrine, and package-boundary discipline.
- Do not preserve the current rendering architecture if it continues to produce the broken public outcome.
- Do not stop at minor code cleanup if the rendered result is still weak.
- The rendered outcome is the primary success measure.

Hard directives

1. Audit the current featured recognition path end-to-end.
2. If the current shared/rendering path is not producing a credible public spotlight, **replace or rebuild it**.
3. Do not continue using a shared surface abstraction merely because it already exists.
4. If the best fix is a new or reworked governed public-surface primitive/family, create it.
5. If the best fix is to move logic out of the current shared surface, do so.
6. The final featured state must be visibly restored and materially different from the current broken screenshots.

Required outcome

The public featured recognition must:
- visibly carry headline/body/recipient/recognition content
- no longer appear as a hollow shell
- feel premium and intentional
- establish the webpart’s quality standard immediately
- justify its final local/shared/ui-kit placement

Required deliverables

- implementation changes
- concise explanation of the root cause
- concise explanation of whether the final solution was:
  - a rebuild in place
  - a replacement of the rendering path
  - a new governed surface
  - or a relocation of logic between local/shared/ui-kit layers
- updated rendered evidence showing the restored featured surface

Acceptance standard

This prompt is successful only if the public featured recognition area is materially rebuilt/restored and the user-visible result is unmistakably better than the current broken state.
```
