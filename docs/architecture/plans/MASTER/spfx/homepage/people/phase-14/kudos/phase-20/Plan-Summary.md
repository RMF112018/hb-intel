# HB Kudos Wave 3 — Plan Summary

## Objective

Correct the full Wave 3 experience-quality issues so HB Kudos becomes more cohesive, more consistently accessible, clearer in interaction behavior, and more reliable in SharePoint-hosted conditions.

## Wave 3 target outcomes

### 1. Make the public experience feel like one product surface
The public HB Kudos experience currently spans multiple surfaces that still feel adjacent rather than fully harmonized:
- masthead / featured
- archive
- article reader
- feed panel
- composer

Wave 3 should improve that cohesion.

### 2. Harmonize flyout and content-body language
“View all”, article reading, archive-related entry points, and composer-adjacent shells should feel more clearly like one family.

### 3. Make accessibility discipline systematic
There is evidence of accessibility intent today.
Wave 3 should turn that into a clearer, repeated system:
- focus-visible behavior
- reduced-motion consistency
- readable control states
- flyout/body structure
- content interaction clarity

### 4. Improve companion control clarity
The companion workspace should retain its operational power while tightening:
- control rhythm
- visual/behavioral consistency
- form and toolbar clarity
- action grouping clarity

### 5. Harden host-safe behavior
Preserve the current SharePoint-hosted protections, but reduce brittleness where possible so the surface behaves more reliably in real host conditions.

## Recommended sequence

1. Lock Wave 3 scope and preserve prior-wave decisions.
2. Harmonize the public experience and flyout/body language.
3. Apply a focused accessibility-system and interaction-clarity pass.
4. Tighten companion controls and harden host-safe behavior.
5. Validate Wave 3 and stop before later-wave closure work.

## What success looks like

Wave 3 is successful only if:
- the public experience feels more unified
- flyouts and content bodies read like one family
- accessibility treatment is more systematic
- companion controls feel clearer and more consistent
- host-safe behavior remains intact and is less brittle
- later waves do not need to revisit basic Wave 3 cohesion work
