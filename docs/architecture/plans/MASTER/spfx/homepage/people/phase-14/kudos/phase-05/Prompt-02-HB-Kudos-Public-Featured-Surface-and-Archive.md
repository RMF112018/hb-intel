# Prompt-02 — HB Kudos Public Featured Surface and Archive

```md
Objective

Conduct a focused UI-only implementation pass on the **public HB Kudos featured recognition and archive/recent-recognition browse experience**.

Primary Intent

The public HB Kudos surface should present:
- one strong featured recognition moment
- one clear subordinate browse path into other recognition items

The goal is not to make both zones equally loud. The goal is to make them work together as a premium recognition experience with clearer hierarchy and better value density.

Critical Constraints

- Use live repo truth.
- Do not begin by re-reading files already in active context or memory.
- Remain strictly compliant with `@hbc/ui-kit` and homepage doctrine.
- Promote or adjust shared UI where appropriate, but do not be mechanical about promotion.
- Keep this focused on the public-facing experience.

Focus

Evaluate and improve:
- featured recognition presentation
- value density of the spotlighted item
- recipient / submitter metadata treatment
- archive / recent-recognition treatment
- archive heading/search relationship
- browse-card quality and subordinate-zone hierarchy

Target Outcomes

The rendered result should move toward the following:
- the featured recognition feels more direct, premium, and content-led
- the archive/recent-recognition area feels intentional rather than leftover
- the browse zone is clearly subordinate to the featured zone but still useful
- value density improves without making the surface feel cramped
- label/heading usage becomes cleaner and less over-explanatory

Implementation Freedom

Choose the best path.

Possible directions may include:
- recomposing the spotlight structure
- simplifying metadata treatment
- changing archive browse format
- rebalancing heading/search placement
- shifting from a blocky archive treatment to a lighter rail or subordinate grid if appropriate

Do not assume one of those directions is mandatory if repo truth suggests a better route.

Do Not

- preserve the current archive structure out of inertia if it remains weak
- keep excessive framing around the featured recognition if it continues to dilute content value
- overcomplicate the browse experience with secondary noise

Deliverables

1. Implement the featured-surface and archive improvements.
2. Update or create shared homepage-safe UI where warranted.
3. Provide a concise summary of what changed.
4. State which remediation-matrix rows were advanced or closed.
5. Explicitly note any remaining archive/composition items deferred to later prompts.

Acceptance Standard

This prompt is successful only if the featured recognition feels more compelling and the archive/recent-recognition experience feels more intentional, subordinate, and better composed.
```
