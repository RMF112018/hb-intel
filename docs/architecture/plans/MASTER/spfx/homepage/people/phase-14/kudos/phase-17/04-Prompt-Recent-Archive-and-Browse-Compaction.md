# 04 — Prompt: Recent / Archive / Browse Rebalance After Compaction

## Mission

Rebalance the **subordinate public-surface zones** after masthead/featured compaction so the entire HB Kudos webpart remains visually coherent and content-carrying.

## Why this prompt exists

The agent must not shorten the hero/featured area and leave recent/archive/feed areas visually oversized, under-authored, or rhythmically disconnected.

## Scope

In scope:
- recent recognition section
- recent rows
- archive section
- archive search
- archive rows
- visible browse/feed rows if the same public-surface language is affected

## Mandatory rules

- Treat this as continuation design work for the same webpart.
- Every affected downstream area must be reviewed relative to the new compacted masthead.
- Conform to ui-kit guidance and existing public-surface grammar.
- Do not turn these zones into generic dense utility lists.
- Improve compactness and authored continuity at the same time.

## Required changes

1. Ensure recent recognition begins early enough to improve first-viewport signal.
2. Tighten recent section spacing and row height as needed.
3. Tighten archive section padding, search-field footprint, and row spacing.
4. Review browse/feed row footprint and adjust if the homepage compaction otherwise makes them feel oversized or mismatched.
5. Preserve interaction clarity for rows and visible controls.

## Explicit instruction

The code agent must address the entire affected subordinate area, not just one row style:
- section paddings
- section labels
- search field height/padding
- row padding
- row gap
- avatar footprint where applicable
- headline/meta spacing
- footer spacing
- any continuation background/panel treatment that now reads too tall after compaction

## Deliverables

- updated recent/archive/browse public-surface treatments
- short note explaining how subordinate zones were rebalanced against the compacted masthead
- screenshots proving the surface reads as one coherent product after the adjustments
