# Prompt 05 — Rendered QA and SPPKG Rebuild

## Objective

Validate that the new dependency-driven design breakout actually changed the rendered homepage in a decisive way, then rebuild the deployable package.

## Required Validation

### A. Visual validation
Confirm the page is now materially less generic in rendered SharePoint-hosted output.

### B. Dependency validation
Confirm the newly adopted packages are:
- actually used
- correctly scoped
- not bloating the package irresponsibly
- contributing to the premium outcome

### C. Rendered evidence
Capture updated screenshots proving:
- the hero is no longer lackluster
- the greeting is integrated into the flagship hero
- launcher, command, and discovery surfaces are materially stronger
- the page no longer reads like Fluent-adjacent webpart cards

### D. Package rebuild
Perform a clean rebuild and produce the updated homepage `.sppkg`.

## Hard Gates

Do not mark this complete if:
- the page still reads as a card grid
- the hero is only modestly improved
- Fluent-like defaults are still visually dominant
- the screenshots do not show a decisive jump in quality

## Required Output

Produce:
- completion note
- dependency usage summary
- before and after screenshot set
- packaging summary
- explicit statement of whether the design box was actually broken

## Important Instruction

Do not re-read files that are still within your active context or memory.
