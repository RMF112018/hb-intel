# HB Central — Design Breakout Dependency Prompt Package

## Purpose

This package exists because the code agent is still trapped inside a **safe enterprise UI box**.

The current behavior is too predictable:

- defaulting toward Fluent-shaped surface patterns
- producing “clean” but generic enterprise cards
- staying inside the common AI-generated UI mold
- optimizing for acceptable instead of memorable
- using SharePoint constraints as an excuse for visual timidity

This package is a **hard corrective**.

Its purpose is to force the agent to:
- stop treating Fluent UI as the visual answer
- adopt a named dependency stack that enables premium visual work
- build a stronger shared visual system before more homepage-specific tweaks
- create a homepage that feels genuinely custom, premium, and productized

## Core Position

Do **not** settle for:
- Fluent-style cards as the dominant homepage surface language
- stock-looking buttons, badges, search inputs, or panels
- AI-default dashboard aesthetics
- thin borders, timid shadows, and polite layout rhythm
- large empty hero slabs with a button and a line of copy

The target is a **truly stunning**, premium HB Central homepage that still respects SharePoint-hosted reality.

## What This Package Changes

This package introduces:
1. an explicit named dependency stack
2. a hard “no default Fluent visual language” rule
3. a premium shared-surface rebuild directive
4. a top-band redesign directive that uses the new surface system
5. stronger instructions for motion, iconography, overlays, depth, and composition
