# Prompt 05 — Final Validation and Closure Report

## Objective

Run the end-to-end validation after the gap-closure work, produce a fresh package, inspect it directly, and generate a final closure report that states what was fixed, what was proven, and what still requires tenant-side proof.

## Prompt

```text
You are now performing final validation for the PnP Operations packaging/render gap-closure effort.

Important working rule:
- Do not re-read files that are still within your active context or memory window unless needed for precision.

Your job:
1. Run the full relevant build/package flow for `hb-webparts`
2. Confirm a fresh `hb-webparts.sppkg` is produced
3. Inspect the package directly
4. Confirm the proof artifacts generated by the hardened pipeline
5. Confirm the packaged PnP Ops runtime matches current repo truth at a meaningful level
6. Confirm the render-path diagnostics and configuration changes are present
7. Identify anything that still cannot be proven without live SharePoint page runtime evidence

Required deliverables:
1. Final report:
   - `docs/architecture/reviews/pnp-ops-gap-closure-final-report.md`
2. If useful, a concise machine-readable validation artifact alongside the report

The final report must contain:
- Objective
- Changes Implemented
- Freshness / Package-Truth Validation
- PnP Ops Runtime / Render Validation
- Remaining Risks
- Exact Tenant-Side Smoke Test Needed
- Final Judgment

The final judgment must answer clearly:
- Is stale app-bundle reuse now blocked?
- Is package/source drift now materially harder to ship?
- Does the final `.sppkg` appear to contain current PnP Ops code rather than older backend-era runtime semantics?
- What exact additional tenant/browser evidence would still be needed for 100% confidence?

Do not hand-wave. Base conclusions on the code changes, the fresh package, and direct package inspection.
```
