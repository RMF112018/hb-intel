## Manifest Scaffold Summary

`ProvisioningManifest` (Step 2 shape):

- `manifestVersion: '0.1.0-scaffold'` (literal type; bumps when the mapper expands).
- `generatedFrom: TemplateSource` — captures `packageName`, `templateName`, `templateVersion`, `contractRef`, optional `sourceCommit`.
- `generatedAt: string` — ISO 8601, supplied via `context.now` in deterministic tests; defaults to `new Date()` otherwise.
- `mutationGate: MutationGate` — frozen, locked, no live mutation, requires human approval.
- `sitePlan` — placeholder declaring `pattern: '/sites/{ProjectBaseNumberNoHyphen}'` and `resolved: null`.
- `objectPlans` — ten slots (`pages`, `libraries`, `lists`, `groups`, `permissionTemplates`, `modules`, `workflows`, `integrations`, `siteHealth`, `provisioningValidation`), each `{ status: 'planned', entries: [] }`.
- `proof` — `{ plannedHash: null, validationStatus: 'planned', noSecretScan: 'planned', noProcoreMirrorScan: 'planned' }`.
- `warnings: readonly string[]`, `blockers: readonly string[]` — empty in scaffold.

The whole manifest object graph is `Object.freeze`d.

## Mutation Gate Summary

- Compile-time: `MutationGate` uses literal `true` / `false` types so a "live mutation allowed" gate cannot be assembled without explicit type widening.
- Runtime: `createFrozenMutationGate()` returns a frozen `{ mutationLocked: true, liveMutationAllowed: false, requiresHumanApproval: true }`.
- Validator: `validateProvisioningManifest` rejects any manifest where the gate triple is wrong, where any prohibited mutation key appears at any depth, where any secret-class key appears, or where any Procore-mirror key appears.
- Public surface: `src/index.ts` does not export any function whose name implies live tenant execution (verified by test).

## Known Limitations

- `objectPlans.*.entries` are intentionally empty; mapping from contract families lands in Step 3.
- `proof.plannedHash` is `null`; deterministic-hash derivation lands in Step 3 with full object-plan content.
- `sitePlan.urlDerivation.resolved` and `sitePlan.title.resolved` are placeholders; resolution lands when Step 3 reads contract source data.
- `MutationGate` literal types do not cover the optional approval-context fields beyond compile-time presence; approval flow is owned by a later step.
- Source-scan tests use substring / regex matching on TS files. They are intentional, pragmatic, and bounded to `src/mapper/**`, `src/contracts/**`, and `src/index.ts` so the validator and guard modules can legitimately reference the prohibited tokens as data.

## Intentionally Not Implemented

- Graph or PnP calls of any kind.
- SPFx wiring, admin UI, deployment changes.
- Procore SDK, Procore HTTP client, secrets handling, or mirror tables.
- Backend or SPFx imports of this package.
