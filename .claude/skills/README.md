# HB Claude Skills Index

## Purpose

Skills are reusable workflows. Use them instead of growing `CLAUDE.md`.

## Skills

| Skill | Use When |
| --- | --- |
| `hb-workspace-surface-router` | First-pass repo surface classification and authority routing. |
| `hb-repo-truth-audit` | Repo-truth audit or exhaustive live repo review. |
| `hb-prompt-package-builder` | Fresh-session prompt, agent prompt, prompt package. |
| `hb-plan-gate-review` | Agent plan or implementation plan review. |
| `hb-post-execution-closeout` | Post-execution report, commit landed, closure summary. |
| `hb-verification-router` | Validation command selection or adequacy review. |
| `hb-sensitive-operation-gate` | Tenant, Azure, Graph/PnP, Procore, CI/CD, app catalog, `.sppkg`, live endpoint, secrets. |
| `hb-spfx-runtime-parity` | SPFx source/build/manifest/runtime/hosted parity. |
| `hb-ui-doctrine-conformance` | UI doctrine, `@hbc/ui-kit`, basis-of-design, premium UI. |
| `hb-doc-classification-gate` | Current-state/normative/historical/deferred/ADR document classification. |
| `hb-doc-authority-cleanup` | Documentation authority, stale docs, source-of-truth cleanup. |
| `hb-platform-primitive-adoption-gate` | Mandatory Tier-1 primitive checks for feature work. |
| `hb-backend-functions-artifact-gate` | Backend Functions route/health/artifact/runtime identity work. |
| `hb-pcc-phase-router` | Project Control Center work. |
| `hb-brand-asset-governance` | Brand assets, logos, fonts, curated web-ready assets. |
| `hb-skill-author` | Create or update Skills. |

## Rules

- Keep Skills workflow-focused.
- Do not duplicate rule bodies.
- Do not use Skills as ceremony.
- Do not use Skills to bypass approval, settings, or security gates.
