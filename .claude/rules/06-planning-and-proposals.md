# 06 — Planning and Proposals

## Primary Rule

Plans must be repo-truth-based, scoped, executable, and non-ambiguous.

## Plan Requirements

For complex work, include:

- objective;
- files/sources to inspect;
- allowed scope;
- forbidden scope;
- implementation sequence;
- validation;
- residual risk;
- closeout requirements.

## Prompt Packages

Prompt packages must instruct the next agent not to re-read files still in its current context unless they changed, final proof needs line-level validation, or scope expanded.

## Better Path Rule

If the user requests a risky or inefficient approach and a safer path is available, propose the safer path plainly.
