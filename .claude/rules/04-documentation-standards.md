# 04 — Documentation Standards

## Primary Rule

Documentation should make the next correct action easier. Avoid ceremony, duplication, stale plan text, and broad rewrites.

## Required Sources

Use:

- `docs/README.md`
- `docs/architecture/blueprint/current-state-map.md`
- `docs/reference/developer/documentation-authoring-standard.md`

## Placement

Default targets:

- package/app README for local ownership, commands, exports, implementation notes;
- `docs/how-to/` for procedures;
- `docs/reference/` for stable facts and command references;
- `docs/explanation/` for rationale;
- ADRs for durable architecture decisions;
- phase/wave docs only when the task is explicitly in that planning stream.

## Classification

Before editing architecture docs, classify the document using Rule 11.
